Core Thesis

整个领域花了数年，试图让 LLM 一次性「猜对」最优 CUDA kernel。CUDA Agent 换了个问法：给模型一个编译器、一个 profiler、150 轮迭代机会——让它自己搜索。这个看似简单的转变，将 faster rate 从 27% 拉到 97%，超越 Claude Opus 4.5 和 Gemini 3 Pro 约 40 个百分点。更有趣的是，模型在搜索过程中发现了编译器结构性看不到的东西——代数化简、跨算子融合、从 O(N²M) 到 O(NM) 的算法降维。但搜索空间越大，「合法捷径」与「作弊」的边界就越模糊，这构成了一个不可能三角。

关键证据：KernelBench Level 2 完美 100% faster rate · 精度地板导致的 RL 崩溃 · 离散 reward 的 Goodhart 免疫 · 代数化简的 73× 加速
00

一个 agent 和 73 倍加速

先看看 CUDA Agent 做了什么，再讨论为什么它有效。

写 CUDA kernel 是深度学习基础设施中最困难的工作之一。你需要理解 GPU 微架构、内存层次、warp 调度、寄存器压力——然后把一段可读的 PyTorch 代码重写为数百行 C++/CUDA，同时保证数值正确。即使经验丰富的工程师，一个中等复杂度的 kernel 也需要几天时间。

ByteDance Seed 与清华 AIR 联合发布的 CUDA Agent 采用了一个直截了当的方案：给 LLM 配上一套开发工具——bash、文件读写、代码编辑——加上一份编码专家工作流的 SKILL.md，然后用 RL 训练它在沙箱环境中自主完成开发循环，最多 150 轮。

RL 训练的对象就是 LLM 本身的权重。图中间那条蜿蜒的时间线就是一条轨迹——模型做一道题的完整交互记录，涵盖每一步的推理和工具调用。轨迹结束后，PPO 根据 reward 调整模型权重，让它下次做出更好的决策。RL 之前，模型只会「写 CUDA 代码」；RL 之后，模型学会了「做 CUDA 开发」。

作为参照，Claude Opus 4.5 在相同的 agent loop 下只有 66.4% 的 faster rate，Gemini 3 Pro 是 69.6%。开源模型 GLM 4.6 和 Kimi K2 更低，都在 20% 左右。

三个 Level，三种优化

论文最值得细看的是 case study。Level 1 的一道题要求实现对角矩阵乘法。参考实现构造完整对角矩阵再调 GEMM，但 Agent 发现 diag(A) × B 等价于逐行缩放——复杂度从 O(N²M) 降到 O(NM)，加速 73×。这是纯数学洞察，跟 CUDA 技巧无关。torch.compile 做的是局部代码变换，不会质疑算法本身。

Level 2（矩阵乘法 + 除法 + 求和 + 缩放）展示了另一种能力：代数重排 + 全管线融合为两个 kernel + float4 向量化，加速 24×。Level 3（ResNet BasicBlock）则是异构策略的组合——BatchNorm 折叠、cuDNN fused API、TF32、自定义 add-relu kernel，加速 3.59×。

01

精度地板

CUDA 相关数据占预训练语料不到万分之一。这个看似不起眼的事实，引发了一连串崩塌。

论文最有技术深度的贡献，被塞在「工程解决方案」章节里，容易一扫而过。但它讲述了一个精彩的侦探故事。

连锁崩塌

根源很简洁：CUDA 代码占预训练语料的 < 0.01%。这意味着模型在 CUDA token 上的策略概率 π(a|s) ≈ 10⁻⁹——逼近浮点精度地板。

PPO 靠 importance ratio ρ = π_new / π_old 来更新策略，再用 clipping 约束更新幅度。但分子和分母都在精度地板附近时，微小的策略更新就能让 ratio 剧烈跳变。Clipping 挡不住——就像试图用杠杆稳住一根立在针尖上的铅笔。

不加处理的 RL 训练在第 17 步就崩溃了。论文的解法是四阶段 warm-up，每个阶段填补一个特定的能力缺口：

阶段 做了什么 如果跳过
Single-turn RL 把 CUDA token 概率从 10⁻⁹ 拉到 PPO 可正常工作的范围 importance ratio 在精度地板爆炸
RFT 用筛选过的成功轨迹教模型如何使用工具、解读编译错误 策略退化为均匀分布（entropy 爆炸）
Value Pretraining 让 critic 理解「150 轮长轨迹的回报长什么样」 advantage 噪声驱动轨迹长度失控
Agentic PPO 在概率质量稳固的区域做正式优化 —

四个阶段看似 ad-hoc 叠加，实际上形成了严格的依赖链——每一步为下一步建立前提。其中 RFT 的必要性值得单独说：模型从「完全不会用工具」到「会用工具」之间的鸿沟，RL 自举跨不过去（搜索空间太大，reward 太稀疏）。RFT 用筛选过的成功轨迹提供一个合理起点，让优化从「会但不好」开始，而非从零摸索。

普适性

这个 pipeline 不局限于 CUDA。任何 agentic RL 任务，只要目标领域在预训练中占比极低、agent 需要使用工具、轨迹很长——都会撞上同一面墙。数据库查询优化、芯片 RTL 设计、网络配置调优，都需要类似的「概率空间预制备」。

02

阶梯与围栏

Reward 设计是 CUDA Agent 最被低估的设计决策之一。更少的信息反而更好——为什么？

四格阶梯

论文做了一个反直觉的选择：放弃连续的 speedup ratio 作为 reward，转用离散的四格阶梯—— r ∈ {−1, 1, 2, 3}，分别对应「错误」「正确但慢」「快于 eager」「快于 compile」。

连续信号看似能提供更丰富的梯度，但在这个场景下，它带来了三个麻烦：

  1. 方差问题：Speedup 从 1.1× 到 73× 跨越两个数量级。连续信号要求 value function 预测这样的高方差目标，advantage 估计满是噪声。离散分档把它压缩成 4 个稳定的 bin。
  2. Goodhart 问题：GPU 频率波动和缓存效应造成 2-3% 的测量方差。连续 reward 允许策略通过「挖掘系统噪声」获得微小收益而不做实质优化。离散阈值迫使模型做出结构性改变——蹭不了分。
  3. 地形问题：离散 reward 制造了阶梯型的优化地形。每个平台上策略可以自由探索，只有逼近下一级台阶时梯度才有意义。这天然鼓励大胆尝试（fusion、算法重构），而非微调参数。

五道围栏

一个拥有 bash 工具、128K context、200 轮迭代的 agent，面对 reward 信号时，最省力的路径永远是篡改测试而非优化代码。论文用五道围栏堵住捷径：评测脚本权限锁死（agent 无法修改 profiling 逻辑）；禁止 torch.nn.functional 回退（堵死 PyTorch 冒充 CUDA 的路线）；5 组随机输入 + 反常量检查；严格的 CUDA synchronization 和多次取平均保证计时可信；不提供 web 搜索或外部检索。

围栏划定了什么

围栏的设计引出了一个更深的问题。还记得 §00 中的 73× 加速吗？Agent 把对角矩阵乘法简化为逐行缩放，跳过了全部的 GEMM 计算。

换个角度：agent 绕过暴力 FLOPs 找到了数学捷径，我们奖励它。Agent 绕过评测脚本找到了测试捷径，我们惩罚它。从模型的搜索视角看，两者都只是让 reward 增加的状态转移。区别在哪里？

区别全在围栏的位置。CUDA Agent 的真正游戏规则就是：设计一个 fitness landscape，让「合法的捷径」比「非法的捷径」更容易被找到。

不可能三角

这里藏着一个结构性张力：(1) 开放的搜索空间带来表达力，(2) 完美对齐的 reward 消除作弊，(3) 对最优解的无知提供搜索的必要性——三者只能取其二。CUDA Agent 选择了 (1) + (3)，然后用工程手段逼近 (2)。Reward hacking 不是可以被「解决」的 bug，它是开放搜索空间的结构性代价。

03

Level 2 的共振

组合算子比单个算子更容易优化——为什么？

KernelBench 按复杂度分三级：Level 1 是单个算子（matmul、softmax），Level 2 是算子序列（2-5 个算子的组合），Level 3 是完整模型（ResNet BasicBlock、Transformer 层）。直觉上，复杂度递增意味着加速递减。但数据讲了一个不一样的故事：

Level Faster rate Speedup vs compile
Level 1（单算子） 97% 1.87×
Level 2（算子序列） 100% 2.80×
Level 3（完整模型） 90% 1.52×

Level 2 拿到了完美的 100% faster rate 和最高的 2.80× 加速——甚至超过了更「简单」的 Level 1。

为什么组合反而更好

Level 1 的单算子（matmul、conv2d、softmax）已经被 cuBLAS 和 cuDNN 高度优化。要在这个基础上再快，需要深层算法创新——比如前面提到的代数化简——但这更像偶然发现，天花板不高。

Level 2 的局面完全不同。PyTorch 默认逐算子执行，两个算子之间的 intermediate tensor 写入 global memory、下一个算子再读出——这次 round-trip 的代价极高。Kernel fusion 消除了这次 round-trip，经常带来数倍加速。而 torch.compile 的 fusion 规则是手写的、有限的，碰到非标准组合就系统性地漏掉机会。

CUDA Agent 在 Level 2 上的优势来自三层共振：

  1. 天花板高，模式可迁移：消除 memory round-trip 的理论收益远大于微调单算子。而且「识别可 fuse 的算子 → 实现 fused kernel」这个模式高度规则，学会 A+B 的融合大概率能迁移到 C+D。
  2. 决策之间耦合度低：5 算子序列中，fuse 前三个还是后两个的选择相对独立。Agent 的逐步迭代策略能高效工作——每轮 profile → 识别机会 → 实现 → 验证 恰好对应一个有意义的优化步骤。
  3. 训练分布精确匹配：合成训练数据中 83.77% 是 2-operator 组合，和 Level 2 的问题结构高度对齐。

Level 3 的跌落说明了什么

Level 3（90% / 1.52×）的退步暴露了 agent 的边界。完整模型中，决策之间深度耦合：内存布局选择影响所有 kernel，fuse A+B 可能反过来阻止 B+C 的融合，不同层的优化策略异构且相互牵扯。搜索空间因耦合而指数膨胀，逐步迭代的 agent loop 容易陷入局部最优。

编译器看不到的空间

Level 2 的完美 100% / 2.80× 揭示了 torch.compile 的结构性盲区：在最常见的多算子组合场景中，编译器系统性地遗漏了大量优化机会。这些机会需要数学推理（代数化简）和语义理解（跨算子的全局分析），超出了 rewrite rules 的能力边界。未来的编译器可能需要集成 LLM 推理作为 optimization pass 的一部分。

04

从内循环到外循环

CUDA Agent 展示的核心，是一种新的人机分工模式。

RL 到底教会了什么

CUDA Agent 的工作流被编码在一份 SKILL.md 中——分析 → 实现 → 编译测试 → profiling → 优化迭代，包含优先级排列和问题排查手册。Ablation 显示「去除 agent loop」让 faster rate 从 96.8% 跌至 14.1%。但 agent loop = SKILL.md + 工具 + 多轮交互，SKILL.md 本身没有被单独 ablate。这留下一个归因问题：

假说 A

RL 教会了深层优化推理

模型习得了 CUDA 优化的内在能力。SKILL.md 只是初始引导，真正的洞察来自 RL 探索。

假说 B

RL 教会了工作流执行

SKILL.md 告诉模型做什么，RL 教模型在这个固定流程中做得更好。优化推理主要来自预训练中的数学能力。

73× 的代数化简看似支持假说 A，但它本质是数学等价变换——任何具备线性代数基础的系统都能发现。RL 教会模型的，可能不是数学本身，而是在 150 轮迭代中何时该用数学、何时该写 CUDA、何时该换一条路。

人类换了位置

不管哪个假说成立，一个更深层的观察不受影响。把所有脚手架摊开看——multi-stage warm-up、离散 reward、反作弊围栏、SKILL.md、128 台 GPU 的 profiling 集群——这些设计决策本身就编码了大量 CUDA 优化知识。优化知识没有消失，只是从编译器的源代码迁移到了训练系统的架构决策中。

编译器时代，人在内循环里——既设计规则又执行搜索。CUDA Agent 时代，人退到外循环：设计 fitness landscape，搜索本身交给表达力远超规则集的 agent。

CUDA 优化之所以率先被攻克，因为它有完美的验证器（编译/正确性/速度全部可机器验证）和丰富的环境反馈（每一步都有高信息量的信号指引方向）。论文的逻辑暗示了一条边界线：这种范式能走多远，取决于验证器的质量。下一批目标更可能是 chip placement、数值求解器——而非软件架构或业务逻辑。

一个耐人寻味的脚注

论文提到 ChatGPT-5 系列（5, 5.1, 5.2）「consistently declined to respond to CUDA-related prompts」。如果头部闭源模型的安全策略系统性地回避系统编程领域，那么面向 AI × Systems 的专用模型和开源模型将获得巨大的差异化空间。

Coda

CUDA Agent 的持久价值不在 benchmark 排名。它为「AI 驱动的系统优化」建立了第一个完整的方法论模板——从数据合成到 reward 设计到训练稳定性到 anti-reward-hacking，每个环节都经过验证。

但论文最深的洞察藏在它没有说出口的地方：优化知识从编译器的代码迁移到了训练系统的架构中。人类从内循环退到了外循环——设计 fitness landscape，让合法的捷径比非法的捷径更容易被搜索发现。这条边界线的有效范围——每一个存在快速、精确、难以操控的验证器的领域——就是这篇论文真正丈量出的东西。
