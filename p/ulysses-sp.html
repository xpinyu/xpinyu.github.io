Ulysses Sequence Parallelism, Intuitively

How long-context training distributes the sequence across GPUs—and why attention briefly needs a different layout.

pinyu 2026.07 DeepSpeed-Ulysses
The idea

Keep the sequence split by tokens for most of the layer. Right before attention, reorganize so each GPU holds the full sequence for a few heads. Run attention locally. Switch back. Two all-to-alls. Everything else is ordinary single-GPU work.

01

The sequence will not fit

Not the weights. The activations along time.

Data parallel splits the batch. Tensor parallel splits layer width. Pipeline parallel splits model depth. None of them solves a different failure mode: one sample whose context is too long for a single device.

What fails is activation memory along the sequence. Each layer keeps hidden states, residuals, and intermediates for every token. That cost scales with L. FlashAttention removes the giant L × L score matrix, but it does not remove “carry a state for every position through the stack.” Past a point, one GPU runs out of room for those activations.

So you need to distribute the sequence itself.

02

The token cut almost works

Most of a transformer layer is local to each position.

Give GPU 0 tokens 0 … L/P − 1, GPU 1 the next span, and so on. Each device stores only L/P positions. Call this the token cut.

For QKV projections and the FFN, the cut is both correct and cheap. Those ops are position-wise. A token on GPU 0 does not need another GPU’s tokens to compute its own query or its feed-forward path. No communication is required.

Attention is different. A query at position t must score against keys from the full allowed context, then mix values from those positions. If a GPU only holds a slice and never communicates, attention is incomplete—not approximate, incomplete.

All-gathering the full sequence before attention restores correctness. It also restores the footprint you left: full L on every GPU at the most expensive moment. The token cut’s memory win disappears right where you needed it.

The tension
Token cut saves memory but breaks attention. Full gather fixes attention but burns the savings. You need full-sequence attention without every GPU holding every channel of the full sequence.
03

Heads are already independent

You need a second axis that is legal to split. Multi-head attention already provides one.

Multi-head attention does not run one fat attention over the full hidden width. It projects each token into Q, K, and V, then splits those vectors into H heads of width d_h (d = H · d_h). Each head computes its own attention:

O_h = softmax(Q_h K_hᵀ / √d_h) V_h

Results are concatenated and passed through W_O. Inside the softmax, head 0 never mixes with head 1. They only meet after concat, through a linear map.

That independence is usually sold as a modeling feature. It is also a systems fact: if two computations do not interact, different devices can own them. When the token cut breaks attention, you are not out of options. You still have heads as a legal partition—usable only for the attention step, while the rest of the layer stays token-cut.

04

Ulysses stays in the token cut most of the time. Before attention, an all-to-all switches to the head cut. Each GPU runs a standard attention kernel on its heads—full sequence, exact math. After attention, a second all-to-all restores the token cut so out-projection and FFN continue on the cheap layout.

Nothing in the attention formula changes. Only who owns which cells changes.

06

Attention rewrites values. All-to-all rewrites seats.

Do not conflate the two operations.

For each head, attention keeps shape and changes meaning:

Q : [L, d_h]  →  O : [L, d_h]

Each row is now a context-mixed vector. Residual connections still line up because the width is unchanged.

Right after the kernel, the layout is still the head cut. Cells hold O instead of Q/K/V, but the same GPUs still own the same heads over the full sequence. Attention did not restore the token cut. The second all-to-all does that, so concat, W_O, and FFN can run on the memory-efficient token shards again.

token cut  →  local QKV, split heads
all-to-all →  head cut
attention  →  cells become O (still head cut)
all-to-all →  token cut
local: concat, W_O, FFN

Those two all-to-alls are the only Ulysses collectives per layer. The attention kernel itself is a normal local call—FlashAttention or otherwise—on data the first all-to-all arranged for it.

07

Limits that belong to this idea

The head cut is free only while heads last.

In the head cut, each rank needs whole heads. So the Ulysses degree P cannot exceed the head count. With grouped-query attention, the tighter bound is often the number of KV heads. Eight KV heads means pure Ulysses tops out at 8 unless you combine it with another sequence-parallel method.

Also: sequence parallelism shards activations along time. It does not shard model weights. Large parameters still need their own strategy. Long context and large models are different scarcities.

08

What to keep

Three moves, in order.

Token-cut the sequence so long activations fit on many GPUs for position-local work. When attention needs the full sequence, switch to a head cut instead of gathering every head. Switch back when local ops resume.

Attention changes what each cell means. All-to-all changes which GPU holds it. That separation is the whole method.

One sentence

Ulysses is temporary head-sharding for exact full-sequence attention, between two all-to-alls, inside an otherwise token-sharded layer.
