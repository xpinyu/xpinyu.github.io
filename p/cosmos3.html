核心逻辑链

训练物理 AI 智能体需要两种能力：理解（从观测推断语义与动力学）和生成（预测世界的未来演化与行动后果）。过去它们被训成两批互不相干的模型——视觉语言模型负责理解，扩散模型负责生成，VLA 负责动作。

Cosmos 3 的赌注：这不是两类任务，而是同一个因果结构的两个方向。动作是因，世界状态变化是果；理解是从果推因，生成是从因推果。如果把语言、视觉、音频、动作全部编码成 token 排进一条序列，那么"理解 vs 生成"就退化为一个简单问题：哪些 token 是已知的（干净），哪些 token 是待预测的（加噪）？

答案是一个双塔 Mixture-of-Transformers：Reasoner 塔做因果自回归，Generator 塔做扩散去噪，两塔只在共享注意力里相遇。六种看似无关的任务——问答、文生图、视频续写、正动力学、逆动力学、机器人策略——全部是同一条 token 序列上干净/噪声模式的不同排列。

1

先看清它要消灭的东西

旧范式的问题不是模型不够强，而是知识被切成碎片，每片都要单独收集数据、单独训练、单独维护。

一个饭后收拾餐桌的家用机器人，在旧范式下需要三个模型接力：视觉语言模型认出杯盘并规划，VLA 生成动作序列，世界模型模拟后果做评估。每个模型有自己的训练数据、自己的表征空间、自己的失败模式。更糟的是它们互相看不见对方学到的东西——世界模型懂重力，但这个知识漏不进规划器。

旧范式：碎片管线
三套权重、三套数据管线、知识互不流通。接口处全靠人工拼接。
vs
Cosmos 3：一套权重
理解重力学到的表征，生成视频时直接可用；策略学到的动作先验，模拟时直接可用。

这不只是工程上的省事。报告里有一个直接证据：把 Generator 的理解塔从通用 Qwen3-VL 换成在物理 AI 数据上训练过的 Cosmos3 Reasoner，机器人域的生成质量分数从 66.5 涨到 71.3（+4.8），自动驾驶域 +2.3——而生成器本身的参数、数据、算力完全没变。理解侧学到的物理常识，透过共享注意力直接变成了生成侧的质量。碎片管线里这种迁移是不存在的。

于是真正的问题变成：

真正的问题

自回归（下一个 token）和扩散（迭代去噪）是两种完全不同的生成数学。如何让它们共享一个 transformer，而不互相干扰？

2

双塔 MoT：一个注意力，两套参数

Mixture-of-Transformers 的答案：每层装两套 LayerNorm 和 MLP，注意力操作符共享但注意力掩码不对称。

输入序列被切成前后两段。前半段叫 AR 子序列：语言 token，加上经 ViT 编码的图像/视频 token——这是"思考"用的。后半段叫 扩散子序列：经 VAE 编码的图像/视频 token、音频 token、动作 token——这是"做梦"用的，训练时被加上噪声，模型要学会去噪。

每个 transformer 层里有两套独立参数：AR token 走 Reasoner 塔的 LayerNorm、注意力投影和 MLP；扩散 token 走 Generator 塔的另一套。两套都从同一个预训练视觉语言模型的权重初始化——所以 Generator 塔白捡了语言模型全部的世界知识，不用从零学"什么是杯子"。

关键在注意力。两塔在注意力里相遇，但相遇方式是不对称的：

双流联合注意力（报告 Eq. 7–8） 
# AR 段：标准因果自注意力，只看自己段内、只看过去
O_AR = Attn_causal(Q_AR, K_AR, V_AR)

# 扩散段：全双向注意力，键值是两段的拼接
O_DM = Attn_full(Q_DM, [K_AR ; K_DM], [V_AR ; V_DM])

读法：扩散 token 可以随便看语言提示和其他扩散 token（双向），所以视频第 8 帧在去噪时能参考第 12 帧，时间一致性就是这么来的。但 AR token 永远看不到扩散 token——条件通路保持纯粹的因果性，语言生成的能力一个字节都不会被污染。

下面这张注意力掩码图是整个架构最浓缩的表达。点播放键，看信息被允许怎么流动：

注意力掩码：谁可以看谁

6 个 token 的迷你序列：3 个 AR（2 个语言 + ⟨BOG⟩）+ 3 个扩散（2 个带噪视频 + 1 个带噪动作）。绿色 = 允许注意，红色 = 被掩码挡住。行是提问者（Q），列是被看者（K）。

允许注意 被掩码
为什么不对称是必需的
如果允许 AR token 看扩散 token，语言建模的目标函数就会被扩散噪声污染——"下一个词"的预测会依赖一堆还没去噪的连续向量，预训练 VLM 的推理能力立刻退化。反过来，扩散 token 看 AR 是零成本的：语言是条件，条件本来就该被看见。不对称掩码让两种数学各取所需。

两塔的分工可以总结成两张卡片：

Reasoner 塔

处理 AR 子序列。下一个 token 预测，因果掩码。训练目标是理解与推理——问答、空间定位、时序推理、动作理解。

输入语言 + ViT 视觉 token
目标next-token prediction
掩码因果（只看过去）

Generator 塔

处理扩散子序列。流匹配目标，预测去噪速度，双向掩码。训练目标是合成——像素、声音、动作轨迹。

输入VAE 视觉 + 音频 + 动作 token

注意最后那行高亮：训练 Generator 时 Reasoner 是冻结的。语言能力是存量资产，训练生成时一分钱不花、一点险不冒。这个"轮流冻结"的编排，是统一模型不翻车的第一道保险。

3

干净的与噪声的：六种任务，一种句法

这是全文最重要的一个认知转换。任务不是由架构决定的，是由 token 的干净/噪声排布决定的。

报告用 v 表示干净的视觉 token，ṽ 表示加噪的。一条序列永远是 [AR 前缀, 扩散段]，扩散段里"干净的在前、带噪的在后"。文生视频是 [S_AR, ṽ₁:N]；图生视频是 [S_AR, v₁, ṽ₂:N]——区别只是第一帧是干净还是带噪。任务切换不需要改任何架构，只需要改排布。

这个句法在动作模态上发挥出真正的威力。动作 token aₜ 被定义为从状态 vₜ₋₁ 到 vₜ 的因果变量——是世界发生变化的原因。现在，把"哪一段干净、哪一段带噪"当旋钮转，三种经典问题就自动浮现了：

同一个数据样本，三种学习任务

一段机器人操作录像：帧 v₀ → 动作 a₁ → 帧 v₁ → 动作 a₂ → 帧 v₂。绿色 = 干净（已知条件），红色 = 带噪（预测目标）。切换下面的模式，注意变化的只有 token 的颜色——架构一动不动。

时间轴 →
干净视频帧 v（条件） 带噪帧 ṽ（待去噪） 干净动作 a（条件） 带噪动作 ã（待去噪）

这就是"世界模型"和"策略模型"是同一个东西的确切含义：给动作预测未来（正动力学），给未来反推动作（逆动力学），两者都预测（策略）——是同一条序列上条件方向的三种取法。一个联合训练的模型在三种模式间共享结构：报告在 PushT 上做了消融，联合训练三模式的模型，逆动力学误差比单模式专训低 72%，策略覆盖率从 74.1% 升到 77.3%。

同样的句法一路推广，就是 Cosmos 3 的全部任务清单：

任务序列排布谁干净 / 谁带噪
语言问答（VLM）[S_AR]只有 AR 段，扩散塔不激活
文生图[S_AR, ṽ]语言干净，图像全噪
文生视频+音频[S_AR, ṽ₁:N, s̃]视频、音频全噪
图/视频生视频[S_AR, v₁:P, ṽ_P+1:N]前 P 帧干净，后续带噪
视频迁移[S_AR, v_ctrl, ṽ₁:N]控制信号（深度/边缘/地图）干净，RGB 带噪
正动力学[…, v, a, ṽ]动作干净，未来帧带噪
逆动力学[…, v, ã, v]帧干净，中间动作带噪
策略[…, v, ã, ṽ]动作与未来帧都带噪，联合生成

动作的统一几何

还有一个藏在动作模态里的设计，值得单独说。自动驾驶的方向盘、相机的推拉摇移、机械臂的关节、人手的抓握——控制空间完全不同，怎么放进同一个模型？Cosmos 3 的答案是：全部翻译成几何语言。任何动作都被分解为至多三个成分：自身位姿（ego pose）、末端执行器位姿（effector pose）、抓握状态。位姿变化不直接记录控制器指令（PID 参数、关节电流这类东西被刻意丢掉），而是记录相邻两帧的相对位姿变换 ΔT = T⁻¹ₜ₋₁·Tₜ，用 3 维平移 + 6 维旋转表示。

于是无人车是 9 维（只有 ego pose），单臂机器人 10 维（ego + 夹爪开合），双臂 20 维，人形 29 维。不同长度的向量通过各自领域的投影矩阵 W_in⁽ᵏ⁾ 映射进同一个隐空间，共享 MoT 主干。翻译层很薄，但这就是"跨本体迁移"在工程上成立的原因——相机运动学到的"位姿变化如何改变画面"，和机械臂学到的，在几何上是同一门语言。

4

把三种时钟拧到同一根轴上

视频 24 帧每秒、音频 25 token 每秒、动作 15 赫兹——联合生成时，位置编码必须按物理时间对齐，不能按 token 序号。

标准的位置编码给每个 token 一个序号：第 0 个、第 1 个、第 2 个。这在纯语言里没问题。但联合生成视频+音频+动作时，"第 5 个 token"在三种模态里对应完全不同的物理时刻——视频第 5 个 latent 帧是 0.83 秒处（24 FPS 经 4 倍时间压缩），音频第 5 个 token 是 0.2 秒处。如果按序号对齐，模型学到的时间对应关系就是错的。

Cosmos 3 改造了 3D MRoPE：时间轴不再数 token，而是数物理时间。每种模态有一个"每秒步数"（TPS），位置增量按 δt = TPS_base / TPS 缩放。基准是视频的 6（24 FPS ÷ 4 倍压缩）。于是 30 FPS 的视频每帧走 0.8 格，16 FPS 的每帧走 1.5 格——同样的真实时长，占据同样的位置区间。

FPS 调制：位置间距反映真实时间

切换帧率，观察同样 5 帧在位置轴上占据的区间如何缩放。16 FPS 的 5 帧覆盖更长的真实时间，所以间距更宽。

消融实验：只用文本告诉模型目标帧率，运动保真度 0.72；只用位置调制，0.74；两者叠加 0.76——文本通道和位置通道学到的是互补的约束。

这个位置编码章节里还埋着一个非常诚实的工程细节。扩散 token 的时间索引本可以从 AR 段末尾紧接着编号——但这样最后一个语言 token 和第一帧视觉 token 的时间嵌入几乎相同，模型分不清"语言结束了、画面开始了"，大模型上会出现画面过饱和和棋盘格伪影。修复方法朴素得可爱：在位置空间里硬插一个 15000 的空档。不加参数、不改架构，只是在"语言"和"画面"之间留出一段位置上的缓冲带，过渡信号立刻变清晰。这类"位置编码当设计变量用"的手法，是这份报告里反复出现的味道。

5

训练课程：先做梦，再接触地面

Reasoner 和 Generator 走两条不同的课程表，但编排原则相同：先通用，再专门，而且每一步都不烧掉上一步的资产。

Generator 的三阶段课程是一个清晰的递进：

阶段学什么数据规模（Nano / Super）
预训练图、视频、音频的通用生成先验；多分辨率混训（256p/480p/720p），token 打包进固定 7.4 万 token 的窗口31.05T / 17.86T token
中训练引入动作（25%）和视频迁移（25%）；针对性补物理 AI 稀缺场景——长时程交互、精细机器人动作、安全攸关驾驶2.4T / 1.9T token
后训练小数据精调成专才：文生图 checkpoint、图生视频 checkpoint、DROID 机器人策略约 500 亿 token（策略）

两个细节值得记住。其一，中训练才引入动作数据——先学会世界怎么演化，再学动作怎么驱动演化，顺序反了会让动作先验没有世界模型可以附着。消融证实了中训练的价值：从"见过动作的中训练 checkpoint"出发做下游适配，在相机运动、驾驶、机器人、第一人称全部四个域上都优于从预训练 checkpoint 出发，差距还不小（机器人正动力学 PSNR 23.24 → 25.52）。

其二，机器人策略后训练时，模型预测的不只是 32 步未来动作，还有辅助的 RGB 未来画面。动作和它的视觉后果在同一个前向过程里被联合去噪——策略网络内部自带一个世界模拟器，这就是论文标题里 "world-action model" 的实体含义。推理时为了速度可以跳过视频解码，只用 4 步扩散采样动作，两块 RTX Pro 6000 就能跑 15Hz 闭环控制。这个 checkpoint 在 RoboArena、RoboLab、MolmoSpaces 三个榜单上同时登顶。

一致性证据
怎么验证"策略预测的动作"和"策略想象的画面"真的互相一致？把预测的动作块拿到 RoboLab 仿真器里执行，对比真实 rollout 和模型想象的画面：第三人称视角 PSNR 23.19 dB——和专门的前向动力学模型相当。想象与行动在同一个表征里对齐，不是两个碰巧同居的子网络。
6

三个不显眼但重要的点

音频教视频学物理

消融里最反直觉的一笔：预训练加入音频后，纯视频指标也变好了。机器人域 T2V 从 81.9 涨到 83.8，AV 域从 64.7 涨到 67.9——评测里根本不含音频。合理的解释是：碰撞有声音、材质有声音、运动节奏有声音，联合建模迫使模型把这些物理属性的表征磨得更准，视频侧白捡了这份约束。模态之间不是竞争参数，而是互相提供监督信号。

理解塔是生成质量的免费杠杆

前面提过：换一个更懂物理的理解塔，生成器不动，物理域生成质量就涨。这反过来说明统一架构里最值钱的设计不是"能同时做很多任务"，而是理解侧的每一点进步都会自动补贴生成侧。你优化 VLM 的推理能力，视频生成就跟着变好——碎片管线里这是不存在的复利。

token 打包是个被低估的工程决策

多分辨率混训的通常做法是按分辨率分桶、牺牲利用率。Cosmos 3 直接把不同分辨率、不同长度的序列打包进固定 7.4 万 token 的窗口，零填充。这个决定听起来纯工程，但它让"同一 batch 里图像、256p 视频、720p 视频共存"成为常态——分辨率无关的表征不是设计出来的，是被数据装载方式逼出来的。配比 1:1:2:1（图像 : 256p : 480p : 720p），480p 占双份，是保真度与样本多样性的权衡点。

7

这个模型的边界在哪

统一不是免费的。知道它在哪里吃力，才算真正理解了它的形状。

  • 通用性输给专才的上限。联合训练在单指标上仍可能略输单模式专训（PushT 消融里正动力学 PSNR 26.22 vs 27.13）。统一买到的是迁移与复用，不是每个单项的极限。
  • 因果性是被架构锁死的。AR 段永远看不到扩散段，意味着"边生成视频边用语言自我解说"这类交错推理不在当前句法内——语言只能在开头当条件，不能在生成中途插入。
  • 动作翻译层丢弃了控制器细节。统一几何表示刻意丢掉 PID、关节电流这类低层接口信息。换来跨本体迁移，代价是部署时仍需领域特定的后处理（DROID 策略用了 Franky 控制器执行预测出的关节位置）。
  • PSNR 是低保真度代理。报告自己承认：生成式 rollout 本来就不必逐像素复现唯一的真实未来。PSNR 高只说明时序对齐和重建保真，不等于物理正确——人评（Cosmos-HUE）和闭环成功率才是更硬的指标。
  • Edge 变体掉队了。从零训练的 2B Edge 模型在数学和科学上碾压同级别对手（HMMT 76.3 vs 22.9），但通用域 MMLU-Pro 反而更低——课程表的取舍在通用知识上付了税。
迁移检验

读完之后，一个问题可以检查你是否真的拿到了这个模型的心智图：

如果要做"视频配音"——输入无声视频，生成匹配的音效——在 Cosmos 3 的句法里，token 应该怎么排？哪个塔先被训练、哪个塔提供条件？

（答案的形状：视频 VAE token 干净、音频 token 带噪，排在扩散段内"视觉→音频"的顺序里，[S_AR, v₁:N, s̃]。条件通路和正动力学是同构的——干净的东西在前，待生成的东西在后，模式只是又一次翻转干净/噪声的排布。）

这就是 Cosmos 3 真正交付的东西：任务空间被压缩成了排布空间。架构只设计一次，新任务只是新排布。
