# 用 Midjourney 做品牌图，真正拉开差距的不是 prompt，而是流程

大多数人用 Midjourney 做品牌图，都卡在同一个地方。

不是不会写 prompt。

而是能偶尔出一张好图，却怎么也做不出一整套风格统一的品牌视觉。

单张图做得再好看，也不是品牌。品牌是一百张图放在一起，别人一眼就能认出"这是你"。

这也正是 AI 创意总监 Jamie Ganon 在 How I AI 播客里拆开的那个问题。她每天可能生成上千张图，给客户交付的不是单张作品，而是一整套可复用的视觉系统。

她的核心判断只有一句：

**一致性不靠手感，靠流程。**

---

## 第一步：用图说话，别用文字

大多数人一上来就开始写 prompt。

但 Midjourney 是视觉模型。你给它看的东西，比你写给它的东西强得多。

所以第一步不是打字，而是去 Pinterest 或 Cosmos 上做一个 mood board。

Jamie 的做法很直接：把你想要的那种"感觉"拼成一面图墙。她做的是一种偏粉色、偏酷、有点互联网编码感、有点反差感的美学，所以 mood board 上会有打了耳钉的橘子、趴在电脑前的荧光水果狗、脏脏的独角兽。

关键不是每张图都完美匹配你的最终目标。

关键是它们加在一起，构成了一种"视觉语言"。

主持人 Claire 说了一句很准的话：

**一张图对 LLM 来说，字面意义上等于一千个词。**

如果你不是设计师、不是摄影师，你可能根本没有足够的专业词汇去精确描述你想要的风格。但你不需要有。你只需要能"指着一张图说：就是这个感觉"。

mood board 就是你的视觉词汇表。

---

## 第二步：把 mood board 变成 Style Reference

mood board 在 Midjourney 里可以直接用，但它有一个问题。

当你的 mood board 里图片风格差异太大时，Midjourney 会试图"取平均值"。最终输出的结果，既不像 A，也不像 B，而是一种模糊的中间态。

Jamie 发现，把同样的图片改成用 Style Reference（SRF）的方式喂进去，效果往往好得多。

SRF 告诉 Midjourney 的不是"参考这个构图"，而是"参考这个整体风格"：色调、对比度、相机处理方式、那种说不清但看得出的"氛围"。

更重要的是，SRF 让你可以做减法。

她举了一个很典型的例子。mood board 里有一张绿色眼影的照片。用了 SRF 之后，生成出来的图片全都偏绿。

她怎么解决的？

不是去 prompt 里写 `--no green`。

而是直接把那张绿色照片从 SRF 里删掉。

还有一个更漂亮的操作。有张参考图里的女孩在吹泡泡糖，泡泡糖太抢眼，导致生成结果全在模仿那个元素。Jamie 的做法不是试图用文字去排除泡泡糖，而是直接在 Midjourney 的 UI 里截图、裁掉泡泡糖那部分，把裁完的图重新拖进去当参考。

一个截图操作，解决了你可能花半小时 prompt 也解决不了的问题。

这里真正值得记住的原则是：

**当某个视觉元素太强势，与其用文字去对抗它，不如直接从源头移除它。**

---

## 第三步：用个性化代码注入你自己的风格

到这一步，你已经有了不错的风格方向。但可能还缺一种"属于你自己"的感觉。

Midjourney 有一个叫 Personalization Code 的功能。

它的原理很简单：给你看大量图片，你选喜欢的，跳过不喜欢的。做完之后，系统会生成一个风格代码，你以后可以随时调用。

Jamie 给自己做了一个叫"late 2025 aesthetic"的代码。她在评分时专门挑那些有 iPhone 质感、现代感、高清感的图片。

加上这个代码之后，之前还差一口气的生成结果，一下子有了更多"深度"：皮肤质感更真实，色调更当代，整体观感从"AI 生成的好看图片"变成了"看起来像真实拍摄的品牌素材"。

不过她也提到了一个陷阱：风格渗透。

如果你在评分时点了太多文艺复兴风格的图，即使你喜欢的只是它们的色彩，生成结果也可能开始往油画方向跑。

所以做 Personalization Code 时要清楚自己在选什么，以及为什么选。

---

## 第四步：用"捷径词"代替长 prompt

到了需要写 prompt 的阶段，Jamie 的原则反而是——越短越好。

她用的最核心的技巧，是用一个词代替一整段描述。

**杂志名和出版物名，就是最强的风格快捷方式。**

比如她写 `Daze editorial photo shoot`。Daze 是一本很有风格的杂志，以 ASAP Rocky 的封面闻名，视觉上高对比、粗粝、街头感极强。这些你可能需要十几个形容词才能说清楚的东西，一个杂志名就全给了。

同理，写 `Vogue` 就自动带来了高级时装、精致打光、优雅构图的全套预设。

**相机型号也是捷径。**

不用记 aperture、ISO 这些参数。她有一个自己整理的相机清单。想要 90 年代数码感？直接贴一个 Sony RX100。想要胶片感？换一个对应的胶片相机型号。Midjourney 的训练数据里有大量用这些相机拍摄的照片，它自然知道那种"味道"。

还有一个非常精巧的例子：她想生成一只鹿在纽约高层公寓里的画面，但不想写一堆描述来说明"这是一栋很高的、有大落地窗的、位于曼哈顿的后现代公寓"。

她只用了一个词：`luxury`。

因为 AI 的训练数据里，"luxury apartment" 就等于高层、大窗、城市天际线、富人住在顶楼。一个词，搞定所有场景设定。

Jamie 自己的总结很直接：

**你的 prompt 应该越来越短，而不是越来越长。**

如果你去 Midjourney 的 Explore 页面看别人的 prompt，很多长得像论文。但真正出好图的人，往往用的是最简洁的人类语言，加上精准的视觉参考。

---

## 第五步：把最终结果打包成"品牌代码"

这一步是整个流程里最容易被忽略的，也是 Jamie 认为最有价值的。

当你已经生成了一批满意的图之后，她会做一件事：

用这批图本身，再做一个新的 mood board。

然后把这个 mood board、对应的 SRF、Personalization Code、以及最终使用的 prompt 结构，全部打包在一个 Figma 文件里交付给客户。

这意味着客户拿到的不只是一组图片。

他们拿到的是一套"配方"——知道用什么参考图、什么风格代码、什么 prompt 结构，就能继续生成同风格的新图。

这完全颠覆了传统的创意服务模式。

过去，品牌方找创意公司，拿到一批照片，用完了就得再花钱买。

现在，Jamie 的交付方式是：**我花大量精力帮你找到你的视觉空间，然后把钥匙交给你。**

客户可以自己继续生成。如果风格需要演进或者遇到问题，再回来找她做下一轮。

这不是在让自己变得不重要。

恰恰相反，这让她可以把精力全部集中在最有价值的部分：定义风格，而不是重复执行。

---

## 加分项：用 Nano Banana 当 AI 版 Photoshop

Midjourney 生成的图再好，有时候细节还是需要修。

手指多了一根。电脑屏幕长得像 2005 年的 ThinkPad。

Jamie 的做法是把图拿到 Flora 或 Nano Banana 里做精修。

她对 Nano Banana 的定位很明确：**它就是 Photoshop，只不过你用语言代替了鼠标。**

比如她有一张很喜欢的图，里面的女生在用一台奇怪的电脑。她直接告诉 Nano Banana：

> Replace the computer she's typing on with a 2026 Midnight Black MacBook Pro. Keep the position and the size of the computer exactly the same. Only the left side and the keyboard is visible.

Nano Banana 是推理模型，知道 MacBook Pro 长什么样，不需要额外给参考图。同时，它还会把图片分辨率从 800×800 拉到 4000×4000。

一次操作，同时完成细节修正和画质升级。

---

## 当你卡住的时候

最后一个值得带走的建议，不是技巧，而是心态。

Jamie 被问到"当 AI 不给你想要的结果时怎么办"，她的回答不是"换一个 prompt"。

她说：**先走开。**

你在里面泡太久，眼睛已经不准了。

回来之后，做一件事：诚实地诊断到底是什么在出问题。

是 SRF 里某张图太强势？删掉它。

是 mood board 的风格太分散？重新整理。

是 prompt 塞了太多东西？砍掉大半。

她用了一个很到位的表述：

**要对自己的生成结果残酷地诚实。**

不是"这张图单独看还挺好的"就够了。

而是"这张图放在品牌组合里，真的像吗？"

如果答案是不像，就不要骗自己。

---

整套流程拆开看，其实就是五个动作：

1. **用 mood board 建立视觉语言**，而不是一上来就写 prompt。
2. **用 Style Reference 注入风格**，用减法去掉干扰元素。
3. **用 Personalization Code 加入你自己的审美偏好。**
4. **用捷径词缩短 prompt**：杂志名、相机型号、一个精准的形容词。
5. **把最终结果打包成可复用的品牌代码**，交付的是配方，不只是成品。

这套方法真正厉害的地方，不是它让生成变快了。

而是它让一致性变得可控了。

当然，这里也有一个边界必须说清楚：

**流程解决的是"风格统一"的问题，不是"审美品位"的问题。**

如果你的 mood board 本身就没有方向感，SRF 和 Personalization Code 只会把混乱放大，而不会让它变好。

所以在走完这五步之前，最值得投入时间的，可能是第零步：

**去大量地看、大量地存、大量地训练自己的视觉品味。**

Pinterest、Cosmos、Twitter 上的审美账号、高级时装摄影、独立杂志，这些都是你的训练数据。

AI 能帮你执行，但决定方向的，始终是你的眼睛。
