# 从“图像像文本一样切块”到“图文共同推理”：ViT、DiT 与 MMDiT 的一条演化线

把 Transformer 从语言搬到视觉里，难点并不只是“把 CNN 换掉”这么简单。真正卡人的地方有两个：第一，图像不是天然的离散 token 序列；第二，图像生成模型怎么“听懂”文本，尤其当文本和图像本来就不是同一种表达形式时。

这也是这段内容最有价值的地方：它不是把 ViT、DiT、MMDiT 当成三种并列名词来介绍，而是把它们串成一条连续的设计演化线。你会看到，很多后来显得“理所当然”的结构，其实都在回答同一组问题：**图像怎样变成 Transformer 能处理的序列？条件信息又该怎样进入这个序列，而不是被它淹没？**

**核心问题：** 当输入从文本变成图像、当条件从类别变成自由文本时，Transformer 的接口和条件注入方式该如何随之升级？

## 一眼看懂
- 这是一段**架构讲解型**内容，主线不是罗列模型，而是说明视觉 Transformer 如何一步步解决“token 化”和“条件控制”两个核心难题。
- **ViT 的关键突破**：不直接让像素互相注意，而是把图像切成 patch，当作 token 喂给 Transformer 编码器。
- **DiT 的关键变化**：输入不再是原始图像，而是扩散模型中的 latent；同时模型必须显式接收“时间步”和“条件类别”。
- **DiT 真正重要的设计点**不是“用了 Transformer”，而是“条件信息怎么注入”：拼接 token、交叉注意力、特征调制、残差门控，各自对应不同代价与控制力。
- **MMDiT 的升级**：不再把文本当作静态提示附加给图像，而是让图像 token 和文本 token 在联合注意力中对称地彼此影响，从而更适合处理歧义和编辑类任务。

## 核心逻辑链
- 图像不像文本那样天然是离散序列，所以想把 Transformer 用到视觉上，第一步必须先解决“如何把图像 token 化”。
- 直接在像素级做自注意力计算量会爆炸，于是 ViT 用 patch 代替像素，把“全图互看”变成可计算的序列建模。
- 到了扩散生成，模型除了处理图像内容，还必须知道自己处于哪个去噪时间步、要生成什么内容，因此 DiT 的核心问题变成“如何把条件信号稳定而高效地注入 Transformer”。
- 简单拼接条件 token 虽然能工作，但条件可能被忽略；交叉注意力更直接但更贵；特征调制和残差缩放则提供了更经济、更细粒度的控制方式。
- 当条件从单个类别升级为自由文本时，把文本当作静态外部输入会暴露局限，尤其在图像编辑这类任务中；MMDiT 因而转向图文对称、联合注意的多模态处理。
- 这套思路的边界在于：Transformer 并不是“天然适合视觉”，它是通过 patching、latent 表示和条件注入机制这些接口改造之后，才变得可用且有效。

## 这位讲者真正想做的事

这段内容表面上是在介绍三个模型：ViT、DiT、MMDiT。可更深一层看，讲者其实在做一件更有教学价值的事：**把它们解释成一系列几乎不可避免的设计选择**。也就是说，不是“研究者发明了很多技巧”，而是当你认真面对视觉输入和生成条件这两个问题时，某些结构会自然冒出来。

所以他先回到 Transformer 的原始形态：机器翻译里的 encoder-decoder。这个回顾不是铺垫背景知识而已，而是在帮你看清一个关键分叉：语言常常按从左到右处理，因此 decoder 的因果注意力很自然；图像则更像一个整体，任何区域都可能帮助理解另一个区域，所以视觉任务通常更偏向使用**双向注意力的 encoder 一侧**。

从这里开始，整个叙述的节奏就很清晰了：先看最简单的视觉任务——分类，于是得到 ViT；再往前走一步到图像生成，于是得到 DiT；再把条件从固定类别升级为自由文本，并要求图文真正交互，于是走到 MMDiT。这不是三次平行的介绍，而是一条“接口不断升级”的连续轨道。

## 核心思想一：视觉版 Transformer 的第一步，不是“看图”，而是先把图像变成 token

**最重要的变化，是把图像从连续像素场改写成 patch 序列。**

这一步今天看上去很自然，但在最初并不明显。Transformer 原本处理的是 token 序列，而图像是一个连续的二维信号。更麻烦的是，如果你天真地把每个像素都当成 token，再做全局自注意力，复杂度会直接失控。

讲者给了一个很具体的量级对比：一个 **256×256** 的小图像，如果每个像素都和所有像素互相注意，计算量会到**几十亿级别**。而 ViT 的做法是把图像切成 **16×16 的 patch**。这样一来，原本的像素网格变成了 patch 网格，token 数大幅下降。

> 作者的那句概括很传神：**“一张图像值 16×16 个词（An image is worth 16x16 words）。”**

在讲者举的这个例子里，图像被切成 **256 个 patch**。这些 patch 先被展平，再经过线性层映射到 Transformer 隐藏维度，比如 **1024 维**，并补上位置编码。于是，图像终于有了一个与文本序列足够相似的“接口”。

这为什么不显然？因为很多人对视觉模型的默认想法是：图像就该按局部卷积来处理，先看边缘，再看纹理，再看物体。而 ViT 的真正赌注是：**只要你把图像切成合适大小的 token，Transformer 的全局关系建模能力就能直接接管视觉任务。**

接下来发生的事也顺理成章：既然是分类任务，那就像 BERT 一样，加一个 **CLS token**。由于编码器是双向注意力，CLS 能看到所有 patch，最后变成整张图的总结，再接一个分类头输出类别分布。

这一步的意义不只是“分类效果更好”。它定义了后面所有视觉 Transformer 的基础接口：**图像不再是像素矩阵，而是 patch token 序列。**

## 核心思想二：到了扩散生成，图像 token 化还不够，模型还必须知道“现在在去噪过程的哪一步”

**DiT 和 ViT 的根本区别，不是任务从分类变成生成，而是模型开始在一个“逐步去噪”的局部过程里工作。**

讲者这里快速回顾了扩散模型：从训练数据中的干净图像开始，逐步加高斯噪声，直到接近纯噪声；训练时，模型学习反过来走这条链，从噪声一步步还原出有意义的图像。

但他强调了一个很容易被忽略的点：在实际系统里，扩散通常**不在像素空间做**，而是在更紧凑的 **latent 空间**做。借助一个外部的 VAE，原图会被压缩成更小的三维张量。例子里，空间分辨率被压到 **32×32**，通道数从 3 变成 **4**，整张图只需大约 **4000 个值**来表示。

这一步不是枝节，而是 DiT 成立的前提。因为如果直接在原始像素空间上用 Transformer 去做扩散，成本会很高。latent 空间既保留足够信息，又把计算规模降到了可接受范围。

接着，DiT 沿用了 ViT 的 patch 思路，只不过切的不是 RGB 图像，而是 latent 张量上的 **3D patch**。讲者给的例子里，总共切成 **64 个 patch**，再展平、线性投影，形成 Transformer 输入。

这里真正重要的新增变量，是**时间步**。因为扩散模型在反向去噪时，每次只做一步局部修正；模型并没有“一眼看完整条链”的全局视野。所以如果你不把当前时间步明确告诉它，它根本不知道现在面对的是几乎纯噪声，还是已经接近成图的 latent。

这也解释了为什么 DiT 的接口和 ViT 不同：
- 输入：从原始图像变成 noisy latent
- 输出：形状与输入对应的去噪结果，或一个与之等价的辅助量（比如预测噪声）
- 条件：除了图像本身，还必须显式输入**时间**，以及最初版本中的**类别标签**

这一步挑战了一个常见直觉：很多人以为“生成模型就是更复杂的分类模型”。但在讲者这里，二者的差别更本质：**分类是在整体理解后输出一个标签；扩散生成是在一个多步局部过程里，不断根据当前噪声水平修正表示。**

## 核心思想三：DiT 的真正设计焦点，是“条件信息怎样进入网络”，而不是“有没有条件”

**给模型条件很容易，难的是让条件真正起作用，而且成本可控。 [▶ 9:49](https://www.youtube.com/watch?v=jNPmSjrxGtY&t=589s&end=647)**

讲者花了相当多篇幅讲 DiT 的四种条件注入方式，这其实是整段内容里最值得保留的部分。因为这里暴露了一个常见误解：不是把时间和类别喂给模型就够了，关键是它们以什么方式影响图像 token。

### 1）最直接的办法：把时间和类别当成额外 token 拼进去
这方法“技术上可行”。条件 token 与图像 patch 一起进入自注意力层，因此理论上可以影响所有图像 token。

但问题也很直接：**它们会和图像 token 争夺注意力带宽**。时间和类别虽然重要，却可能在海量图像内容中被忽视。

这是一种非常典型的“能用但不稳”的方案。

### 2）更明确的办法：增加交叉注意力
这里不再是所有 token 混在一起自注意，而是让**图像 token 作为 query，条件 token 作为 key/value**。这样每个图像 patch 都被迫去“看”条件信息。

这解决了“条件被忽略”的问题，但代价更高。讲者提到，这一额外注意力模块会增加大约 **15% 的 gigaflops**，是四种方案里最贵的。

### 3）更巧的办法：用特征调制
这是讲者解释得最精彩的一段，因为它不是凭空出现，而是从 2017 年的 **FiLM** 工作一路接过来的。

FiLM 当年处理的是视觉问答。核心想法是：CNN 里不同 feature map 关注图像的不同方面，有的看轮廓，有的看细节。问的问题不同，就该强调不同的 feature map。于是用问题 embedding 生成每个 feature map 的 **缩放参数 γ 和偏移参数 β**，对特征进行调制。

DiT 把这个思想挪到了 Transformer 里。 [▶ 12:43](https://www.youtube.com/watch?v=jNPmSjrxGtY&t=763s&end=822)因为每个 patch 的 embedding 是一个长度为 **1024** 的向量，那么跨所有 patch 看，第 1 维就是一张“feature map”，第 2 维又是一张，直到第 1024 维。于是，时间和类别条件就可以生成 **1024 维的 γ、β 向量**，对这些“特征切片”做调制。

这为什么特别适合扩散？讲者给了一个非常直观的解释：  
- 在 **t=1**、噪声很高时，模型更该关注**高层结构**
- 接近 **t=0** 时，模型才更该关注**细节修饰**

也就是说，时间不仅是“一个额外标签”，它还应该改变模型内部到底强调哪些特征。

这就是 DiT 中所谓的 **adaLN（adaptive layer norm）**：在注意力块和前馈块之前，用条件信号动态生成 scale 和 shift，对激活进行调制。

这一步挑战的默认想法是：条件只需要在输入层告诉模型一次。DiT 给出的更强观点是：**条件应该深入网络内部，持续改变各层看待特征的方式。**

### 4）最后一步：让条件控制残差分支的强弱
这是讲者补上的稳定训练机制。思路来自更广泛的经验：在训练初期，如果高层随机参数的影响太强，学习会更难。一个有用的做法是在残差分支前加一个缩放参数 **α**，并把它初始化为 0。这样训练刚开始时，上层几乎被“静音”，网络等效地更浅，优化更稳定。

DiT 又把这个缩放做成条件相关的函数，让 **α 由时间和类别决定**。于是条件不仅能调制特征，还能控制当前层激活究竟该放大多少。

讲者特别指出，时间条件在这里尤其有意义。因为时间与噪声强度、也与任务难度相关，所以它可以帮助模型根据当前噪声水平，**柔性地调节网络的“有效深度”**。

把这四种方式放在一起看，DiT 的重点就清楚了：它不是简单地“把 Transformer 用于扩散”，而是在认真寻找**条件如何以不同力度、不同成本渗透进网络**。

## 核心思想四：从类别条件到文本条件，真正的问题不是“能不能加文本”，而是“图文能不能相互解释”

**MMDiT 的升级，来自对“静态文本条件”局限性的正视。**

最初的 DiT 只是**类别条件**：比如猫、狗这样的固定类目。后来一些工作把它扩展到文本提示。讲者提到一个例子：**PixArt-alpha** 基于前面第二种方案，也就是交叉注意力，把单个类别 embedding 换成了由外部语言模型 **T5** 产生的一串文本 token embedding。

这当然能工作。但讲者认为它有点“不够理想”，因为这些文本 token 是**静态的**：它们被文本编码器算出来之后，就那样放着，并不会被当前图像影响。

这在图像编辑任务里会很受限。讲者给的例子非常好：

> “把球棒（bat）移到右边”  
> 在这张图里和在另一张图里，指的可能根本不是同一个对象。

也就是说，文本的意义并不总是自足的；它常常依赖图像上下文来消歧。

MMDiT 的做法因此更对称：图像和文本各自有**独立的编码参数**，但它们通过**联合注意力（joint attention）**沟通。做法是把图像 patch token 和文本 token **拼成一个大序列**，然后对这个序列做自注意力。

关键在于这种对称性：
- 图像 token 不只看图像 token，也看文本 token
- 文本 token 也不只保留原本的语言表示，它同样会看图像 token

这意味着两种模态不是“一方解释另一方”，而是在同一个注意力空间里**共同澄清彼此的含义**。

讲者还补了一个实现层面的细节：联合注意力模块本身不额外拥有一套新的模态无关参数；query、key、value 仍由各自模态的编码器分别计算。联合注意力做的事情，是在拼接后的序列上计算并应用注意力分数。之后输出再被分流回图像侧和文本侧，继续沿各自编码路径前进。

这一步带来的理解升级是：**多模态条件不是把一种模态附着在另一种模态上，而是让它们在表示层里彼此塑形。**

## 串起来看

从 ViT 到 DiT，再到 MMDiT，这条线表面上是在讲三个架构，底层其实只围绕两个问题反复展开：**如何把视觉输入变成 Transformer 可处理的 token 序列；如何让条件信息有效进入并改变这个序列的处理过程。**

ViT 解决第一个问题：图像切块。  
DiT 把这个接口带入扩散生成，并把“时间”提升为不可缺少的条件变量，随后认真探索条件注入机制。  
MMDiT 则进一步承认：当条件变成自由文本时，真正需要的不是更多提示词，而是图文之间的双向对齐与共同消歧。

这也是讲者最想传达的总图景：这些架构不是“技巧堆叠”，而是随着任务要求升级，接口与条件机制被一步步逼出来的结果。

---

## 最小必要心智模型
- **建模对象是什么：** 用 Transformer 处理视觉数据时，核心问题是把图像或 latent 变成 token 序列，并让外部条件有效影响这些 token 的更新。
- **它怎么运作：** ViT 先用 patch 把图像序列化；DiT 把同样思路搬到扩散 latent 上，再通过时间、类别或文本条件去调制注意力、特征和残差流。
- **它在哪失效：** 如果条件只是被简单附加、却不能持续而明确地参与表示更新，它就可能被忽略；而当文本意义依赖图像上下文时，静态文本编码也会变得不够用。

## 一句话记住
视觉 Transformer 的演化，本质上是在不断升级两个接口：**图像怎样变成 token，条件怎样真正进入 token 之间的计算。**

# 不是“Transformer 接管视觉”，而是视觉被迫发明了两个新接口

**我：**  
我先说个很粗的理解：ViT、DiT、MMDiT，不就是 Transformer 从文本一路扩到图像分类、图像生成、图文生成吗？

**苏格拉底：**  
这话不算错，但太像“一个锤子找到了更多钉子”。我先问你一个更扎人的问题：如果你把原版 Transformer 直接拿来喂图像，第一下会卡在哪？

**我：**  
图像不是词的序列？

**苏格拉底：**  
对，一半。另一半是：就算你硬把每个像素当一个 token，算力也会先爆炸。材料里举了个很具体的量级：一张 **256×256** 的图，如果每个像素都跟所有像素做自注意力，计算量会到**几十亿级**。这还只是“小图”。

所以这条演化线，先不是“Transformer 会不会看图”，而是两个更朴素的问题：

1. **图像怎么变成它吃得下的 token 序列？**  
2. **条件信息怎么进来，且别被图像本身淹没？**

先把这两个接口看清，后面三个模型就不会像三张背诵卡片。

---

**我：**  
那 ViT 本质上就是在回答第一个问题？

**苏格拉底：**  
正中。ViT 干的最关键一刀，不是“取消 CNN”，而是说：**别按像素看，按 patch 看。**

材料里那个口号你应该记住：  
**“一张图像值 16×16 个词。”**

意思不是诗意，而是工程。把图像切成 **16×16 的 patch**，原来像素级那种不可算的全局注意力，就被压缩成 patch 级的全局注意力。例子里一张图被切成 **256 个 patch**。这些 patch 被展平、线性投影到比如 **1024 维**，再加上位置编码，终于长得像 Transformer 熟悉的输入了。

所以 ViT 的核心贡献，不是“用 Transformer 做分类”，而是定义了一个接口：

> **图像 = patch token 序列。**

这一步一旦成立，分类只是顺手的事。

---

**我：**  
“分类只是顺手的事”有点夸张吧。分类不是还得有整图理解吗？

**苏格拉底：**  
当然要，但那个问题在 Transformer 里已经有现成零件了。ViT 借了 BERT 的 **CLS token**。把它放进 patch 序列里，靠双向注意力让它看全图，最后把它当作整张图的摘要去做分类头。

注意这里有个容易被忽略的点。讲者专门回顾了 Transformer 原始的 encoder-decoder 结构，不是为了怀旧，而是为了帮你看清一个分叉：

- 语言常常天然适合**从左到右**，所以 decoder 的因果注意力很自然；
- 图像更像一个**整体**，任何区域都可能帮助理解另一个区域，所以视觉更偏向用 **encoder 这一侧的双向注意力**。

所以 ViT 不是“把整台机器搬过去”，而是主要借走了更适合视觉的那半边。

---

**我：**  
好，ViT 我大概明白了：先把图像伪装成序列。那 DiT 不就是把这个办法拿去做扩散吗？

**苏格拉底：**  
“拿去做扩散”这句也对，但又太轻描淡写。DiT 比 ViT 多了一个你不能省略的变量。你猜是什么？

**我：**  
时间步？

**苏格拉底：**  
对。这里就是第一个真正该打开的认知缺口：**为什么扩散模型非得显式知道时间？**

因为分类和扩散的工作方式根本不同。

分类像什么？像看完一张卷子后打一个总分。  
扩散像什么？像修复一张从严重损坏到接近完整的图，每次只修一小步。

材料里说得很直白：在反向去噪时，模型是**每一步被调用一次**，只做局部修正。它并没有“看完整条链”的全局视野。所以如果你不告诉它当前是接近纯噪声的阶段，还是快成图的阶段，它根本不知道该做粗修还是细修。

这就是 DiT 和 ViT 的根本差别之一：  
ViT 处理的是一个静态理解问题；  
DiT 处理的是一个**带阶段信息的多步修正过程**。

---

**我：**  
但时间步不就是再喂一个标签进去吗？为什么它这么重要？

**苏格拉底：**  
因为它不是普通标签，它在扩散里几乎等于“当前任务难度”和“当前该关注哪类特征”。

材料里那个解释很值得记：  
- **t 接近 1**，噪声很大，模型更该抓**高层结构**；  
- **t 接近 0**，图快干净了，模型才该管**细节修饰**。

所以时间不是“附加说明”，而是决定当前这一步该怎么计算的核心变量。

如果你愿意，可以把它理解成医生做手术时的阶段信息：现在是止血、缝合，还是精修外形。你不能说“反正都是一台手术，知道病人是谁就够了”。

---

**我：**  
那 DiT 的输入还是图像 patch 吗？

**苏格拉底：**  
不是原始图像了。这又是一个关键前提，很多人会跳过去：**现代扩散通常不在像素空间里做，而在 latent 空间里做。**

材料里给了个很具体的压缩例子：  
原图先经过外部 VAE，被压成一个更小的三维张量。空间分辨率降到 **32×32**，通道从 3 变成 **4**。整张图大约只要 **4000 个值**来表示。

为什么这一步重要？因为如果你还在原始像素上跑 Transformer 式扩散，成本会很难看。latent 空间相当于先把图像“浓缩”成一个更适合建模的工作介质。

然后 DiT 复用了 ViT 的思路：  
不是切 RGB 图，而是切 **latent 上的 3D patch**。例子里切成 **64 个 patch**，再展平、投影，送进 Transformer。

所以你现在可以先记一个骨架：

> **ViT：把图像切成 patch，好让 Transformer 能读。**  
> **DiT：把 latent 切成 patch，好让 Transformer 在扩散过程中逐步去噪。**

差别不只是任务变了，而是**输入空间**和**关键条件变量**都变了。

---

**我：**  
那我有点疑惑。既然时间和类别都要输入，最简单不就是把它们也当 token 拼进去吗？为什么后面还搞那么多条件注入方式？

**苏格拉底：**  
好问题。这正是整段材料里最值钱的部分。

很多人以为“条件生成”的难点是：**有没有条件。**  
DiT 真正逼你看的问题是：**条件到底以什么方式进入计算，才能真的起作用？**

你刚说的方法，就是第一种：  
把时间和类别当成额外 token，直接拼到图像 token 前面。

这在技术上完全能跑。因为自注意力允许所有 token 互相影响。问题是，它有个很现实的毛病：

> **时间和类别会跟大量图像 token 争夺注意力带宽。**

换句话说，它们可能在理论上存在，在实践里却被“礼貌性无视”。

---

**我：**  
“礼貌性无视”挺形象。所以第二种交叉注意力，就是强迫图像 token 去看条件？

**苏格拉底：**  
正是。材料里说得很清楚：  
图像 token 做 **query**，条件 token 做 **key/value**。这样每个图像 patch 都要去读取条件信息，而不是把条件丢在混杂的人群里等人来搭理。

这当然更稳，但代价是更贵。讲者提到额外的交叉注意力大约会增加 **15% 的 gigaflops**。

所以这里第一次出现了一个很成熟的工程张力：

- **拼接 token**：便宜，但可能被忽略；  
- **交叉注意力**：更直接，但更贵。

看到没有？这已经不是“模型结构花样多”，而是**控制力和计算成本的权衡**。

---

**我：**  
那第三种为什么会更“巧”？

**苏格拉底：**  
因为它不再问“条件要不要被看见”，而是问：**条件能不能直接改模型内部看特征的方式？**

这就是材料里花了不少时间讲的 **FiLM → feature modulation → adaLN** 这条线。

先别急着记术语，我们用人话说。  
FiLM 当年做视觉问答时，想法很朴素：如果问题是“这是什么狗”，那模型该更看重毛发细节；如果问题是“狗在左边还是右边”，模型该更看重轮廓和位置。也就是说，**不同问题应该放大不同特征图，压低另一些特征图。**

于是它用问题 embedding 生成一组参数，对特征做线性调制：  
**缩放 γ，平移 β。**

DiT 把这个思想搬到 Transformer 里时，做了一个很漂亮的对应：

每个 patch embedding 是一个 **1024 维向量**。  
如果你跨所有 patch 去看第 1 维，那是一张“feature map”；第 2 维又是一张；一直到第 1024 维。  
于是时间和类别就可以生成 **1024 维的 γ、β**，去调制这些维度。

这意味着条件不只在输入层说一句“顺便提醒你，我是猫，时间步是 0.7”。  
它是在每一层、每一块里，持续地改模型对哪些特征更敏感。

---

**我：**  
所以 adaLN 本质就是“条件驱动的层归一化参数”？

**苏格拉底：**  
对，这个总结够用了。标准 layer norm 里也有 scale 和 shift，但通常是固定可学习参数；**adaptive layer norm** 的关键是：这些参数是由条件动态生成的。

材料里还提到，DiT 把这种调制放在两个地方：  
- 多头注意力块之前  
- 前馈网络块之前

这透露出一个重要判断：  
**条件不是只该决定“输入是什么”，还该决定“每一层怎么处理输入”。**

这是比“加个条件 token”更深一层的控制。

---

**我：**  
那第四种残差门控，看上去像训练技巧，不像核心思想。

**苏格拉底：**  
它既是训练技巧，也是对“条件注入”更进一步的回答。

先回忆一下残差块：每一层不是推倒重来，而是把当前层的输出加回主干。材料里说，有一个很普遍的经验是：训练初期，上层随机参数的干扰太大时，让它们先安静一点，优化会更稳定。办法是在残差分支前加一个缩放参数 **α**，并让它初始接近 0。这样网络一开始等效更浅。

DiT 又往前迈了一步：  
**α 也由条件决定。**

所以条件现在不只控制“看什么特征”，还控制“这层当前该发多大声”。

尤其时间在这里特别合理。因为时间和噪声强度相关，也就和任务难度相关。噪声很大时，也许网络某些高层分支该更强；接近成图时，又可能是另一套力度分配。

这就有点像一个总指挥，不只是给乐手发谱子，还动态调节每个声部的音量。

---

**我：**  
我好像开始理解为什么你说 DiT 的重点不是“用了 Transformer”，而是“条件怎样渗入网络”。但这是不是有点过度设计？只要交叉注意力够强，不就够了吗？

**苏格拉底：**  
这就是很好的抵抗。我们不妨把问题说得更尖一点：

> **条件信息到底应该像“被访问的外部资料”，还是像“渗进系统内部的控制信号”？**

交叉注意力更像前者：图像 token 需要时去查条件。  
特征调制和残差门控更像后者：条件直接改网络内部的动力学。

哪种更好，不是抽象地说，而要看任务。对于扩散，时间不是可有可无的背景知识，它强烈决定每一步该如何工作。所以让它深度影响内部计算，往往是合理的。

这也是材料真正想教你的：  
**条件注入方式不是装饰，而是在定义条件有多大控制权。**

---

**我：**  
那从 DiT 到 MMDiT，核心升级就是把类别换成自由文本？

**苏格拉底：**  
如果只是“把类别换成文本”，那你会低估 MMDiT 的动机。实际上，材料先举了一个过渡方案：像 **PixArt-alpha** 那样，沿用交叉注意力，把单个类别 embedding 换成由 **T5** 产出的文本 token 序列。

这“技术上可行”。但讲者觉得它**不够理想**。为什么？

因为这些文本 token 是**静态的**。文本编码器先把提示词算完，后面图像只能去读它们；文本自己的表示并不会被当前图像修正。

这在某些任务里没那么致命，在另一些任务里就会暴露硬伤，尤其是**图像编辑**。

---

**我：**  
是因为同一句文本在不同图里意思不一样？

**苏格拉底：**  
正是材料里那个很妙的例子：  
**“把 bat 移到右边。”**

在一张图里 bat 可能是球棒；在另一张图里可能是蝙蝠。  
甚至就算都指球棒，图里到底哪一个物体才是“那个 bat”，也得看图像上下文。

所以问题不再是“文本能不能作为条件输入”，而是：

> **文本的意义能不能被图像现场地消歧？**

如果文本表示永远静态，那它就像一个先写好的法条，到了现场不允许结合案情解释。很多时候这不够。

---

**我：**  
所以 MMDiT 的“多模态”重点不是加两个模态，而是让它们互相改写？

**苏格拉底：**  
说得非常准。

MMDiT 的做法是：  
图像 token 和文本 token 各自有自己的编码参数，但在中间通过 **联合注意力（joint attention）** 沟通。具体做法很朴素：把两边 token **拼成一个大序列**，然后在这个联合序列上做注意力。

关键不在“拼起来”三个字，而在它的**对称性**：

- 图像 token 会看文本 token；  
- 文本 token 也会看图像 token。

材料还特地补了一个实现细节：联合注意力模块本身并不是另起一套全新模态无关参数；query、key、value 还是由各自模态分支算出来。联合注意力主要做的是在拼接序列上计算注意力分数，然后再把输出分流回各自通道继续处理。

这意味着两种模态不是一主一辅，不是“图像是正文，文本是批注”，也不是反过来。它们是在同一个注意力场里**共同澄清彼此的含义**。

---

**我：**  
但听起来它和“交叉注意力双向做两次”有点像。真正差别在哪？

**苏格拉底：**  
你这个问题很好，因为它逼我们别把“联合”误解成一个口号。

一个工作上的区别是：交叉注意力常常保留比较强的主从关系——比如图像端去查询文本端。哪怕你反向也做一遍，它也仍像两个系统彼此访问。  
而 MMDiT 的联合注意力更强调：**它们在同一个序列空间里同时更新，语义是共构的。**

这在处理歧义时尤其重要。因为歧义往往不是“一边给另一边补充资料”就解决的，而是双方表示都要被重新塑形。  
文本“bat”的表示要根据图像变，  
图像里某个细长物体是不是球棒，也可能要根据文本任务目标重新变得突出。

也就是说，解决歧义不是单向检索，而是双向对齐。

---

**我：**  
我想试着自己归纳一下。ViT 解决“图像怎么 token 化”；DiT 在这个基础上加上“时间和条件怎么进入去噪过程”；MMDiT 则处理“当条件是自由文本，而且意义依赖图像时，条件不能再是静态外部输入”。这样说对吗？

**苏格拉底：**  
对，而且已经很接近整条线的最小模型了。不过我还想再逼你一下：这三者表面上是三种模型，底层到底是同一个什么问题在升级？

**我：**  
嗯……是不是同一个 Transformer 接口在被不断改造？

**苏格拉底：**  
更具体一点。

**我：**  
两个接口：  
一个是**数据接口**——图像怎么变成 token；  
一个是**条件接口**——外部信息怎么真正影响 token 之间的计算。

**苏格拉底：**  
这就对了。

把这两个接口抓住，你下次看到新模型时就不容易被名词淹没。你先别问“这是不是又一个某某 Transformer”，先问：

1. **它把视觉输入变成 token 的方法是什么？**  
   是原图 patch？latent patch？更细还是更粗？代价是什么？

2. **它怎么注入条件？**  
   是拼接？交叉注意力？特征调制？门控？联合注意？  
   条件是浅层地被看到，还是深层地改变整个计算流？

这两个问题比背模型名更有迁移性。

---

**我：**  
那这种理解的边界在哪？会不会最后什么都能被说成“接口设计”？

**苏格拉底：**  
这正是要小心的地方。说“接口设计”很容易滑向空泛。你得盯住三个具体边界。

**第一，Transformer 不是天然适合视觉。**  
材料反复在提醒这一点。它之所以能用，是因为有人先发明了 patching、latent 表示这些接口改造。离开这些，原版做法要么不自然，要么算不动。

**第二，条件不等于控制。**  
你把条件喂进去，并不代表模型就会认真用它。拼接 token 能工作，但可能被忽略；这就是为什么后面要比较不同注入机制。

**第三，文本不等于语义已经定了。**  
当文本含义依赖图像上下文时，静态文本编码就会露出局限。MMDiT 的意义在于承认语义是共同形成的，而不是预先封装好的。

你看，这三条都很具体，不空。

---

**我：**  
如果我下次看一个新的视频生成模型，最先该检查什么？

**苏格拉底：**  
很好，开始迁移了。先别急着看它有多少层、多少参数，先查这三件事：

**第一，看它的 token 化对象。**  
是直接在像素帧上做，还是在压缩后的 latent 上做？  
是把每帧独立切 patch，还是把时间维也一起打包成时空 patch？  
这决定了它到底在什么粒度上建模，以及成本大头在哪。

**第二，看条件是什么类型。**  
是类别、时间、文本、动作轨迹、参考图，还是别的视频片段？  
不同条件不是“多加几路输入”那么简单，它们对注入方式的要求不同。

**第三，看条件进入网络的深度。**  
它只是前面拼一下，还是每层都有调制、门控、联合注意？  
如果任务高度依赖条件精确控制，而条件只在浅层出现，那通常值得警惕。

这三个检查项，基本就是你从这篇材料里该带走的工作模型。

---

**我：**  
那我再问一个有点挑刺的问题：是不是可以说，ViT 解决的是“可计算性”，DiT 解决的是“可控制性”，MMDiT 解决的是“可消歧性”？

**苏格拉底：**  
这说法挺漂亮，而且不只是漂亮，基本成立。

- **ViT：**先把全局注意力从像素级的不可承受，改造成 patch 级的可承受。  
- **DiT：**让生成过程不只是“会生成”，而是能被时间和条件稳定牵引。  
- **MMDiT：**让文本条件不只是附着在图像外面，而能和图像一起确定“它到底是什么意思”。

如果你非要压成一句，那就是：

> **视觉 Transformer 的演化，不是在换 backbone，而是在升级两个接口：输入如何序列化，条件如何掌权。**

---

**我：**  
“条件如何掌权”这个说法有点狠，但我记住了。不过还有一个地方我不太踏实：为什么讲者一直强调这些设计像“几乎不可避免”的选择？研究里不是充满偶然和试错吗？

**苏格拉底：**  
当然有偶然和试错。讲者不是在神话历史，而是在做一种很有价值的教学整理：把表面上杂乱的技巧，重新还原成被问题逼出来的选择。

你认真面对图像，就会碰到“token 不天然存在”这个问题。  
你认真面对扩散，就会碰到“时间步必须显式进入”这个问题。  
你认真面对文本条件，尤其是编辑和歧义，就会碰到“静态文本不够”这个问题。

于是某些结构——patch、latent、调制、交叉注意、联合注意——就不再像随机灵感，而像一系列逐步收紧的回答。

这不等于它们是唯一答案，只是说：  
**一旦问题被问对了，很多设计选择会显得没那么任性。**

---

**我：**  
我感觉我现在能带走一个很短的模型了。你帮我看看对不对：

第一步，把视觉对象改写成 Transformer 可处理的 token。  
第二步，别只把条件“放进去”，而要决定它在多大程度上持续改变 token 更新。  
第三步，如果条件本身会被上下文改变含义，就让它和图像一起更新，而不是保持静态。

**苏格拉底：**  
这已经很好了。再把它压成一个你以后真会用的版本：

> **先问序列化，再问条件注入，再问条件是否需要双向消歧。**

这就是最小可迁移模型。

- 遇到 **ViT 类问题**：先看序列化。  
- 遇到 **DiT 类问题**：重点看条件如何渗入层内计算。  
- 遇到 **MMDiT 类问题**：追问条件是不是必须和主模态共同解释语义。

---

**我：**  
那最后留个开放问题吧。如果按这条线继续走，下一个值得警惕的误区会是什么？

**苏格拉底：**  
我给你留一个不只是开放、还能顺手测你是否真懂的问题：

如果明天你看到一个新模型，宣称自己把文本、图像、视频、动作指令全都塞进了一个统一 Transformer，于是“多模态理解与生成大一统”了——

你第一反应不该是惊叹，而该先问：

1. **这些模态是怎么 token 化的？粒度是否合适？**  
2. **不同条件只是被拼接，还是在层内持续掌权？**  
3. **语义歧义是靠静态编码硬扛，还是靠联合更新现场消解？**  
4. **为了得到这种“统一”，它牺牲了哪种专门控制力或计算效率？**

如果你会这样问，你就没把 ViT、DiT、MMDiT 当三个名词背下来；你已经把它们变成了一个能迁移的判断工具。

而真正还没完的问题是：
