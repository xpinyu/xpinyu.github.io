# DiT 为什么有效：把扩散模型的骨架从 U-Net 换成 Transformer 之后，真正变了什么

很多人第一次听到 DiT，会自然地把它理解成一句很简单的话：**“就是把扩散模型里的 U-Net 换成了 Transformer。”**  
这句话不算错，但它其实漏掉了最关键的部分：**DiT 的效果提升，并不只是因为“换了个架构名字”，而是因为它把扩散模型放进了一个更容易随着计算量扩张而持续变强的骨架里。**

视频里有个很重要、也很容易被忽略的细节：在作者的比较中，**同一类模型里，性能和 FLOPs 的相关性很强**。甚至在 patch size 变小、参数量不一定更大时，FID 还是更好。这说明 DiT 的核心不是“参数更多”，而是**可扩展的计算结构**。

先抓住这个最小模型，后面很多设计就都顺了：**DiT 不是单纯把图像生成“Transformer 化”，而是把扩散建模改写成“在潜空间中，对一串图像 patch token 做带条件的去噪预测”，并且这个过程会随着 token 数、层数、宽度和总计算量增加而稳定受益。**

**核心问题：** DiT 到底改了哪些关键部件，为什么这些改动会让扩散模型更适合随着算力扩张而提升表现？

## 核心逻辑链
- 扩散模型原本常用 U-Net，但如果想要更系统地扩大模型规模，Transformer 提供了另一条更规则的扩展路径。
- 为了让 Transformer 处理图像生成，DiT 不直接看像素，而是先用预训练 VAE 把图像压到潜空间，再把潜空间图像切成 patch token 序列。
- 生成任务还需要时间步和条件信息，DiT 比较了多种注入方式，发现用 **adaLN-Zero** 在每层调制归一化和残差缩放，效果最好。
- 当 patch 更小、token 更多，或 Transformer 更深更宽时，模型性能持续提升；这并不主要来自参数量变化，而更像是来自**计算量扩张带来的建模收益**。
- 但这个结论有边界：视频强调的结果主要来自**类条件**设置；换成文本条件时，作者本人也提示**cross-attention 可能更合适**。

## 说话者真正想做的是什么

这段内容本质上不是在兜售一个“Transformer 一定全面碾压 U-Net”的口号，也不是在提出一套全新的扩散训练方法。它更像是在做三件事：**第一，帮你把 DiT 从 ViT 重新推导出来；第二，解释哪些改动是为了适配扩散任务；第三，把这些设计落到可以自己写出来的代码结构上。**

所以视频的重心，其实不是“扩散的基本原理”——那一部分说话者明确说了，训练、采样、noise scheduler 和之前 DDPM、LDM 视频是一样的。真正新增的内容是：**如何把一个本来做分类的 ViT，变成一个在潜空间中做条件去噪预测的生成模型。**

这也是开头那个认知缺口的答案。DiT 的关键不是“把卷积换成注意力”这么简单，而是下面这条完整改写链路：

1. **去掉 ViT 的分类头**  
2. **改到潜空间而不是像素空间**
3. **patchify 潜变量图像，得到 token 序列**
4. **把时间步和类别条件注入 Transformer**
5. **让输出 token 再变回潜变量图像**
6. **最后通过 VAE decoder 回到像素空间**

一旦你按这条链路理解，DiT 就不再像一个抽象名词，而像一个非常具体的系统。

## 核心思想一：DiT 不是“Transformer 直接画图”，而是在潜空间里对 patch token 做去噪

**最重要的转变，是把“图像生成”改写成“潜空间 token 序列的条件去噪预测”。**

视频先从 ViT 复习开始：ViT 把图像切成 patch，每个 patch 投影成一个 D 维 token，再加上位置编码，送进一串 Transformer encoder 层。分类任务里通常还会加一个 CLS token，最后拿它做分类。

DiT 继承的是这个骨架，但任务完全变了。

具体锚点很清楚：作者不是在 **256×256 像素图** 上直接操作，而是先用和 latent diffusion model 相同的预训练 VAE，把图像压成**下采样 8 倍的潜变量图**。所以，256×256 图像会变成 **32×32、4 通道**的 latent。模型的输入不是干净 latent，而是**加噪后的 latent**；输出则是去噪结果或噪声预测。

这个改动为什么重要？因为如果你直接对高分辨率像素做 Transformer，自注意力代价会非常高。潜空间把空间尺寸缩小了很多，Transformer 才变得可行。

这也澄清了一个常见误解：DiT 的“Transformer 化”不是孤立发生的。它和 latent diffusion 的思路是绑在一起的。**如果没有潜空间这一步，整个设计的计算代价和可训练性都可能完全不同。**

它带来的直接后果是：模型真正处理的“图像单位”不再是像素，也不再是 U-Net 里的多尺度特征图，而是**潜变量图上的 patch token**。

## 核心思想二：从 ViT 到 DiT，真正关键的结构改动有三处

**如果把 DiT 看成 ViT 的生成版，那么最关键的新增件其实只有三类：位置编码调整、条件注入、以及输出端的 unpatchify。**

先说 patch embedding。  
视频里讲得很清楚：这一步与 ViT 很像。把潜变量图分成 patch，每个 patch 展平成长度为 `patch_h × patch_w × channels` 的向量，再线性投影到 Transformer 的 hidden dimension。

但位置编码这里有一个具体变化：**DiT 用的是 2D sinusoidal position embedding**，而不是 ViT 里常见的 1D learnable position embedding。  
也就是说，每个 patch 不是只有一个“序列位置”，而是同时有 **x 轴位置和 y 轴位置**。分别做正弦位置编码后，再拼接成一个 D 维表示。

这个点在视频里不是被说成“决定成败的大发现”。相反，说话者很克制：ViT 论文作者也试过 2D 位置编码，但对分类任务没发现特别显著的差异；而 DiT 论文对位置编码消融也没展开太多。  
所以这里最合理的理解不是“2D 正弦编码神奇地提升性能”，而是：**在把二维网格图像转成 token 序列时，DiT 选择了一个更贴近二维结构的位置信息表达。**

第二个结构改动是输出端。 [▶ 7:47](https://www.youtube.com/watch?v=aSLDXdc2hkk&t=467s&end=526)  
ViT 做分类，不需要把 token 再拼回图像；但 DiT 需要。于是作者加了一个 **unpatchify block**：它把每个 D 维 token 通过 MLP 映射回一个 patch 的像素（更准确地说，是 latent patch）维度，再按原来的网格位置拼回一张潜变量图。

这个部分非常关键，因为它说明 DiT 的 Transformer 主干虽然处理的是序列，但它并没有丢掉“图像几何结构”，只是把结构延后到输出重组时恢复。

第三个结构改动，就是下一节要说的条件注入方式。

## 核心思想三：DiT 最重要的工程选择之一，不是“有没有条件”，而是“条件怎么进每一层”

**同样的时间步和类别信息，注入方式不同，效果可以差很多。**

在 U-Net 扩散模型里，时间步通常做成 sinusoidal embedding，再过几层全连接，注入到 ResBlock；文本条件则常常通过 cross-attention 接进来；类条件可能直接加到时间嵌入上。

DiT 没有沿用同一套写法，而是系统比较了 4 种方案。 [▶ 9:45](https://www.youtube.com/watch?v=aSLDXdc2hkk&t=585s&end=645)视频把这部分讲得非常清晰：

1. **In-context conditioning**  
   把时间步和类别表示当成两个额外 token，拼到 patch token 序列前面，让所有 Transformer 层一起处理。

2. **Cross-attention**  
   在 self-attention 和 MLP 之间再加一个 cross-attention 层，让 patch token 去关注由时间步和类别组成的两个条件 token。

3. **Adaptive LayerNorm（adaLN）**  
   把时间步和类别表示相加，再过一个 MLP，预测 LayerNorm 的 scale 和 shift，用它们调制每一层。

4. **Adaptive LayerNorm Zero（adaLN-Zero）**  
   在上面的基础上，再额外预测 attention 输出和 MLP 输出在残差相加前的缩放系数，而且这些缩放在初始化时设为 0。

实验结果里，**in-context 最差**，**cross-attention 和 adaLN 接近**，**adaLN-Zero 最好**。

这里最值得记住的，不是结果排名本身，而是背后的机制直觉。视频用了一个很好的解释：  
在 adaLN-Zero 里，每个 DiT block 一开始几乎是**恒等映射**。因为残差分支上的两个输出缩放参数在初始化时是 0，所以网络刚开始训练时，整层基本不改输入。这和残差网络里用 identity mapping 稳定训练的思想很像。

这个设计为什么不那么显然？因为很多人看到条件生成，会默认觉得“把条件当 token 丢进去，让注意力自己学”就够了。但 DiT 的结果提示了一件事：**对扩散任务来说，条件信息更像是对整个去噪计算过程的逐层调制，而不是仅仅作为几个额外内容 token 被读取。**

> “the entire dit block kind of just acts as an identity function at the start”

这句近乎口语化的话，其实就是这个设计最核心的训练直觉。

不过视频也明确给了边界：这个结论主要是**类条件生成**里的结果。说话者还特别提到，**文本条件**可能不一样，因为文本序列更长、更丰富，cross-attention 很可能更合适；并举了后续文本到图像 DiT 论文采用 cross-attention 的例子。

这点很重要，因为它防止我们把 “adaLN-Zero 最好” 误读成无条件普适结论。

## 核心思想四：DiT 的表现提升，最强信号来自“计算扩展”，而不只是模型参数量

**视频里真正最有解释力的实验结论，是性能更像在跟 FLOPs 走，而不是简单跟参数量走。 [▶ 14:40](https://www.youtube.com/watch?v=aSLDXdc2hkk&t=880s&end=938)**

作者比较了多种模型规模：从较小的 DiT 到更大的 DiT；同时也比较了不同 patch size。这里 patch size 非常值得注意：

- patch size = 8×8 → token 更少，共 16 个 token  
- patch size = 4×4 → 64 个 token  
- patch size = 2×2 → 256 个 token

结果并不意外地显示：**模型更深、更宽时更强；在同样 Transformer 配置下，patch 更小、token 更多时也更强。**

但真正有意思的是后面那层分析。视频指出，如果你仔细看同一规格模型的不同 patch size 版本，**参数量差别并不大**；甚至 patch size 2 的版本参数量还可能略小于 patch size 4 和 8。因为 patch embedding 和 unpatchify 层在 patch 变大时参数会增加。

可性能最好的还是 patch size 2。

那差异从哪来？视频给出的答案是：**FLOPs 显著增加了。**  
token 更多，自注意力和后续计算都更重。 [▶ 15:38](https://www.youtube.com/watch?v=aSLDXdc2hkk&t=938s&end=998)所以从 XL/8 到 XL/4，再到 XL/2，虽然参数量没怎么膨胀，计算量却大幅上升，而 FID 也随之改善。

这就是 DiT 最值得读者更新模型的一点：**在这个架构里，扩展 token 数、层数、宽度带来的总计算量提升，和生成质量存在强相关。**

换句话说，DiT 的价值不只是“Transformer 也能做扩散”，而是：**Transformer 让扩散模型的性能更像语言模型、视觉 Transformer 那样，能比较顺滑地沿着计算规模继续往上推。**

视频提到，最大的 DiT XL/2 在 FID 上甚至优于 latent diffusion models，而且其架构复杂度（以 GFLOPs 衡量）和 LDM 接近。 [▶ 19:32](https://www.youtube.com/watch?v=aSLDXdc2hkk&t=1172s&end=1231)这进一步说明，DiT 至少在论文作者的设置下，不只是一个“风格不同的等价替代品”。

## 核心思想五：实现上最难理解的地方，其实是 adaLN-Zero 的初始化逻辑

**如果你要自己实现 DiT，最容易写错、但又最体现设计思想的地方，就是 adaLN-Zero 怎么让每个 block 在初始时近似恒等。**

视频后半部分走代码时，把这件事拆得很细。

先是 patch embedding：  
用 `einops.rearrange` 把 `B × C × H × W` 的潜变量图，重排成 `B × N × patch_dim` 的 patch 序列，然后线性投影到 hidden size，再加上 2D 正弦位置编码。

再是标准 Transformer 组件：  
注意力层还是熟悉的 qkv 投影 + scaled dot-product attention + 输出投影；MLP 通常是 4 倍隐藏维度；LayerNorm 因为要被条件控制，所以**关闭默认的 affine 参数**，不让它自己学 weight/bias。

重点在条件调制。  
每一层拿到同一个 conditioning 向量（时间步表示和类别表示），通过一个线性层预测出 **6 组 D 维参数**：

- attention 前 LayerNorm 的 scale / shift
- MLP 前 LayerNorm 的 scale / shift
- attention 输出残差缩放
- MLP 输出残差缩放

其中最后两组缩放参数在初始化时设为 0。

而对 LayerNorm 的 scale，代码里不是直接乘 `scale`，而是乘 **`1 + scale`**。这样一来，初始化时 scale=0，就等于乘 1，不会改动归一化结果。与此同时，shift=0，不平移；残差分支输出再乘 0，于是整个块刚开始近似只剩 identity。

这套设计背后其实有一个很朴素的训练哲学：**先让深层网络“不要急着乱动”，再逐渐学会在条件控制下偏离恒等映射。**

视频在最终输出层也沿用了相似思路：  
最终 LayerNorm 同样由条件预测 scale/shift；最后把 token 投影回 `patch_h × patch_w × channels` 时，输出线性层权重也做零初始化。整体上是很典型的“稳定起步”风格。

如果把这个机制拿掉，模型仍可能能跑，但你就失去了 DiT 里最有辨识度、也最有训练意义的一层设计。

## Connecting the Dots

把这些点连起来看，DiT 的真正创新不是单点技巧，而是一种很完整的重写方式：**它把扩散生成从“卷积特征图上的多尺度去噪”，改写成“潜空间 patch token 上、由条件逐层调制的 Transformer 去噪”，并且证明了这个形式会随着总计算量扩张而稳定变强。**  
位置编码、unpatchify、adaLN-Zero、patch size 实验，这些看起来分散的细节，其实都在服务同一个目标：让扩散模型进入一种更标准、更可扩展的 Transformer 计算范式。

---

## 最小必要心智模型
- **建模对象是什么：** DiT 建模的是“潜空间图像的 patch token 序列在扩散时间步上的条件去噪过程”。
- **它如何运作：** 先把 latent 图切成 patch token，用 Transformer 处理，再用时间步和类别信息通过 adaLN-Zero 逐层调制，最后把 token 拼回 latent 图并解码成图像。
- **它在哪里失效或需要谨慎：** 视频中的最好结果主要针对类条件；换成文本条件时，不能直接假定 adaLN-Zero 仍然优于 cross-attention。

**一句话记住**
DiT 的关键不是“用 Transformer 代替 U-Net”本身，而是把扩散去噪改写成一种能随着 token、深度和 FLOPs 持续扩张的潜空间 Transformer 计算。

**苏格拉底：**  
对。这个前提非常重要。  
很多人说“DiT = diffusion + Transformer”，说得像 Transformer 一上来就面对像素海洋。但材料里其实很明确：**它和 latent diffusion 的那一步是绑在一起的。**

你想想看，如果直接在 256×256 像素上切成非常细的 token，自注意力的代价会很快爆掉。先压到 32×32 latent，Transformer 才进入一个能工作的计算区间。

所以这里已经能看到一个很实用的检查点：  
以后你再看任何“Transformer 做生成”的方案，先别问“是不是注意力更强”，先问：**它到底在哪个表示空间里算？像素空间、潜空间、还是别的压缩表示？**

**我：**  
明白了。那从 ViT 变成 DiT，除了输入变 latent，还有什么必须改？

**苏格拉底：**  
别急着列一长串。先抓三件最关键的。

第一，**去掉分类头**。  
ViT 原来是做分类的，有 CLS token，最后拿它做分类输出。DiT 不需要这个，它要输出的是整张 latent 图的去噪结果或噪声预测。

第二，**patchify 和 unpatchify 成了一进一出的骨架**。  
ViT 只需要把 patch 变成 token；DiT 还得把 token 再还原成 patch，再拼回网格。所以视频专门强调了末端的 unpatchify block：先把每个 D 维 token 通过线性层或 MLP映回 `patch_h × patch_w × channels`，再按位置还原成 latent 图。

第三，**条件信息必须进来，而且不是随便进。**  
扩散模型必须知道当前时间步，还常常有类别或文本条件。DiT 的重点之一，就是研究这些条件究竟怎么注入 Transformer。

这三件事里，最容易被低估的是第三件。

**我：**  
我本来还真觉得，时间步和类别条件就当两个 token 塞进去，让注意力自己学，不就行了？

**苏格拉底：**  
这正是材料要纠正的第二个直觉。

视频里作者比较了四种条件注入方式：

1. 把时间步和类别表示作为两个额外 token 拼进去，也就是 in-context conditioning  
2. 用 cross-attention  
3. 用 adaptive layer norm，也就是让条件去调制 LayerNorm 的 scale 和 shift  
4. 用 **adaLN-Zero**

结果是：  
**直接把条件当 token 拼进去，最差。**  
cross-attention 和普通 adaLN 接近。  
**adaLN-Zero 最好。**

**我：**  
为什么“直接塞 token”反而最差？听起来最自然啊。

**苏格拉底：**  
因为“自然”不等于“适合这个任务”。

你把类别和时间步当成两个 token，本质上是在说：  
“这两个条件也是序列中的普通内容，大家自己通过注意力去读它。”

但扩散里的条件，尤其是时间步，不太像一段“内容”，更像是**整层计算该怎么做的控制旋钮**。  
它不是给你补充一点信息，而是在说：  
“现在噪声有多大，你这一层应该用什么方式处理输入。”

这就是 adaLN 系列为什么合理：它不是把条件当附加内容，而是把条件当**逐层调制整个计算的信号**。

视频里那个非常口语化、但非常重要的解释是：  
> “the entire dit block kind of just acts as an identity function at the start”

这句话几乎把 adaLN-Zero 的核心训练直觉全说完了。

**我：**  
我听懂了一半。“逐层调制”我能接受，但“identity function at the start”为什么那么重要？

**苏格拉底：**  
我们把这个拆小一点。

在 adaLN-Zero 里，每一层都会拿到同一个 conditioning 向量——也就是时间步嵌入和类别嵌入融合后的表示。然后通过一个小 MLP，预测出六组参数：

- attention 前的 LayerNorm scale / shift  
- MLP 前的 LayerNorm scale / shift  
- attention 残差分支输出的缩放  
- MLP 残差分支输出的缩放

重点在最后两组：  
**这两个残差缩放在初始化时被设为 0。**

结果是什么？  
刚开始训练时，attention 分支算出点什么不重要，先乘 0；MLP 分支算出点什么，也先乘 0。于是整个 block 一开始几乎不改输入，差不多就是恒等映射。

**我：**  
所以它像在说：先别乱改，先稳住？

**苏格拉底：**  
正是。  
这背后是一种很朴素但很有力量的训练哲学：  
**深网络起步时，先让每层“默认什么都不做”，再慢慢学会在条件控制下做有意义的偏离。**

视频里还提到一个实现细节，你如果自己写代码，很容易写错：  
LayerNorm 的调制不是直接乘 `scale`，而是乘 **`1 + scale`**。

为什么？  
因为初始化时 `scale = 0`，那 `1 + scale = 1`，就不会破坏 LayerNorm 的正常输出；shift 也初始为 0，不会平移；残差输出再乘 0，于是整个 block 初始近似 identity。

这不是漂亮小技巧而已，它几乎就是 DiT 训练稳定性的招牌设计。

**我：**  
那它比 cross-attention 好，是因为更稳定训练吗？

**苏格拉底：**  
至少从材料能支持的范围看，**稳定训练和“条件作为调制而非内容”这两点，是最可信的解释。**

但我们要小心别过度神化。视频也明确给了边界：  
这个结论主要来自**类条件生成**。  
说话者甚至主动提醒：如果换成文本条件，情况可能不同，因为文本不是一个类别 ID，而是一长串丰富 token，**cross-attention 很可能更合适**。他还举了后续文本到图像 DiT 工作继续用 cross-attention 的例子。

所以正确记法不是“adaLN-Zero 永远最好”，而是：  
**当条件比较短、像全局控制信号时，逐层调制可能比把条件当内容 token 更合适。**  
**当条件本身是丰富序列时，cross-attention 往往更自然。**

**我：**  
这就有意思了。也就是说，条件注入方式其实取决于“条件是什么东西”。

**苏格拉底：**  
很好，这已经接近可迁移模型了。

下次你看一个条件生成架构，先不要问“业内最近流行哪种注入法”，先问：  
**这个条件更像全局控制信号，还是像一段需要被查询的内容？**

- 时间步、类别 ID，更像控制信号  
- 一长串文本、更像内容序列  
- 风格向量、相机参数、噪声强度，也常常偏控制信号

这个区分，会比记住某篇论文的胜负表更有用。

**我：**  
好，那我们回到一开始那个最重要的结论：为什么说 DiT 真正厉害的是“扩展性”？只是因为 token 更多时 FID 更好吗？

**苏格拉底：**  
不是“只是”，而是**那组证据非常干净地指向了计算扩展**。

我们把那组实验再咬紧一点。对同一模型规格，比如 XL：

- XL/8  
- XL/4  
- XL/2

Transformer 的大体骨架没变，但 patch size 变小，token 数从 16 到 64 再到 256。  
结果是，**XL/2 最好。**

如果你只看参数量，会困惑：  
“奇怪，参数没明显增多，怎么效果涨这么多？”

视频给出的关键解释是：  
因为 token 更多，**FLOPs 大幅增加了。**  
自注意力和后续计算都更重，所以真正被放大的不是参数库存，而是每次前向在做的计算量。

这说明 DiT 的一个核心特征是：  
**你给它更多计算，它比较会把这些计算转成更好的生成质量。**

**我：**  
可很多模型也会“更多算力更强”啊。DiT 的特别之处是什么？

**苏格拉底：**  
你说得对，几乎所有模型都会从更多计算中受益。特别之处不在“会不会”，而在**关系是不是足够平滑、稳定、可预期**。

材料想传达的，不是一个神话式结论“Transformer 完胜 U-Net”，而是更克制的一点：  
**Transformer 给扩散模型提供了一条更规则的扩展路径。**

这条路径体现在几个维度都能扩：

- 深度更深  
- 宽度更宽  
- token 更多  
- FLOPs 更高

而实验里性能和这些计算尺度的关系很强，尤其和 FLOPs 的关系很强。这就像在说：  
“这套骨架不是只在一个甜点配置上灵，它能沿着算力往上推。”

对于大模型时代，这比“某个小技巧提了 0.4 FID”重要得多。

**我：**  
所以 DiT 的胜负手，不是某个局部技巧，而是：它让扩散模型更像语言模型或 ViT 那样，进入了一个比较顺滑的 scaling regime？

**苏格拉底：**  
对，这句话已经很接近材料精髓了。

不过我还想逼你再精确一点：  
**为什么 patch size 变小这件事，特别说明问题？**

**我：**  
因为它把 token 数变多了，注意力能处理更细的空间单位？所以建模更细？

**苏格拉底：**  
这是一半。另一半是：  
它说明提升未必来自“你塞了更多参数”，而是来自**你让模型在更细粒度的 token 序列上做了更多计算**。

也就是说，DiT 的收益不是简单的“记忆容量更大”，而更像是**计算过程本身更充分**。  
这和只盯参数量的思路不一样。参数量像是仓库大小，FLOPs 更像是你每次实际调动了多少工人、跑了多少工序。

所以如果你以后看类似模型，不要只问“几B 参数”，还要问：  
**token 长度是多少？每层算了多少？总 FLOPs 怎样？**

这在 Transformer 系模型里尤其重要。

**我：**  
那 U-Net 在这里到底输在哪？材料好像也没直接说“U-Net 不行”。

**苏格拉底：**  
对，别替材料说它没说的话。  
视频没有说“U-Net 全面落后”或者“Transformer 普适碾压”。它更像是在说：

1. 过去扩散模型常用 U-Net  
2. DiT 证明：你也可以把扩散写成 latent patch token 上的 Transformer 去噪  
3. 在这个写法里，**随着 FLOPs 扩张，性能提升很稳定**  
4. 在作者的设置下，最大的 DiT XL/2，FID 甚至优于 latent diffusion model，而且 GFLOPs 复杂度接近

这说明 DiT **不只是一个风格替代品**。  
但它没有推出“从今以后 U-Net 都该退休”。

**我：**  
这让我想到一个问题：如果 DiT 的关键是 scaling，那位置编码、unpatchify、这些实现细节是不是就没那么重要？

**苏格拉底：**  
别走到另一个极端。  
它们不一定是“论文成败的最强因子”，但它们是**把系统拼成可运行整体的必要件**。

比如位置编码。视频说得很克制：DiT 用的是 **2D sinusoidal position embedding**，而不是 ViT 常见的 1D learnable positional embedding。  
说话者没有把它吹成秘诀，甚至提醒：ViT 作者当年试过 2D，分类任务里没看到特别大的差异；DiT 论文也没对这个做很多消融。

所以你最合理的理解不是“2D 正弦位置编码神奇提效”，而是：  
**既然输入本来是二维 patch 网格，那用更贴近二维结构的位置表达，是一个自然、干净的工程选择。**

unpatchify 也是一样。它不是“创新点排行榜第一”，但如果没有它，Transformer 只会吐出一串 token，没法还原成 latent 图。

所以这些组件的地位可以这么理解：  
- scaling 是 DiT 最值得带走的结构性结论  
- adaLN-Zero 是最有辨识度的训练设计  
- latent + patchify/unpatchify + 位置编码，是把这套东西真正落地的骨架零件

**我：**  
这样看，DiT 好像是一种“完整改写”，不是单点魔法。

**苏格拉底：**  
正是。我们把它串起来，你看是不是比“换个架构”清楚得多：

- 不在像素空间做，而在 **VAE latent space** 做  
- 把 latent 图切成 **patch token 序列**  
- 用 Transformer 处理这串 token  
- 用时间步和类别条件对每层做 **adaLN-Zero 调制**  
- 把输出 token **unpatchify** 回 latent 图  
- 再用 VAE decoder 回到像素图  
- 并且这个系统会随着 **token、深度、宽度、FLOPs** 增长而稳定变强

这才是完整的 DiT，不是“U-Net → Transformer”那种一行替换。

**我：**  
如果我要自己实现，最容易写错、但又最能体现思想的地方，是哪里？

**苏格拉底：**  
材料给的答案很明确：**adaLN-Zero 的初始化逻辑。**

你要特别记住三个点：

1. **LayerNorm 关闭默认 affine 参数**  
   因为 scale / shift 要由条件来预测，不让 LayerNorm 自己偷偷再学一套。

2. **LayerNorm 的缩放写成 `1 + scale`**  
   这样初始化时 `scale = 0`，相当于乘 1。

3. **attention 和 MLP 的残差输出缩放初始为 0**  
   所以整个 block 起步近似 identity。

另外，视频还提到最终输出层也沿用类似思路：  
最终 LayerNorm 的调制来自条件；最后把 token 投回 patch 的线性层，权重也做零初始化。整个系统都在贯彻“稳定起步”的哲学。

如果你把这些都改成普通初始化，模型未必一定崩，但你就不再是在实现材料里的那个 DiT 了。

**我：**  
我试着总结一下，你帮我挑错。  
DiT 不是“Transformer 直接生成图像”，而是“在 latent 空间把图切成 patch token，然后用 Transformer 做条件去噪”。  
它强的关键不只是架构名换了，而是这种 token 化的 Transformer 计算更适合沿着 FLOPs 扩张。  
条件注入里，类别和时间步更像控制信号，所以 adaLN-Zero 这种逐层调制比把条件当 token 更有效，至少在类条件设置里是这样。

**苏格拉底：**  
不错，我只改两个地方，让它更扎实。

第一，把“更适合沿着 FLOPs 扩张”说得更操作化：  
**当你想判断一个 DiT 变体为什么更强时，优先看总计算量、token 数、深度、宽度，而不是先盯参数量。**

第二，把条件注入的结论加上边界：  
**adaLN-Zero 在材料里的最好结果主要是类条件；文本条件不能直接照搬，cross-attention 可能更合理。**

这两个边界加上，你的模型就更能迁移，不容易乱用。

**我：**  
那我下次再看到类似论文，最先该检查什么？

**苏格拉底：**  
先检查三个变量，顺序都别乱：

**第一，模型到底在哪个空间里计算。**  
像素？latent？别的压缩表示？  
这决定了 token 数、注意力代价和可行性。

**第二，条件到底是什么类型。**  
是类别 ID、时间步、风格向量这种全局控制？  
还是文本、布局、轨迹这种内容序列？  
前者优先想调制，后者优先想 cross-attention。

**第三，性能提升到底跟什么一起涨。**  
是参数量？token 数？FLOPs？训练数据？  
如果作者只给参数量，不给 FLOPs 和 token 长度，你对“可扩展”这个词就该谨慎一点。

这三个检查点，比背论文名字更值钱。

**我：**  
那材料最后让我真正该带走的一句话是什么？

**苏格拉底：**  
带走这个最小模型：

**DiT 的关键不是把 U-Net 换成 Transformer，而是把扩散重写成“潜空间 patch token 上、由条件逐层调制的 Transformer 去噪”，并证明这种写法会随着 token、深度和 FLOPs 增加而持续受益。**

如果你记住这个，很多细节都能自己长回来。

**我：**  
还有什么问题是材料没彻底回答，但值得继续想的？

**苏格拉底：**  
有一个很好的开放问题，而且正好能检验你是不是真的理解了。

如果现在不是做类条件图像生成，而是做一种新任务：  
条件是一段中等长度的结构化序列，比如一串人体姿态点、一个粗布局、或者一小段动作轨迹；输出仍然是图像。  
那你会先押注什么——adaLN 风格的逐层调制，还是 cross-attention？  
以及，你判断的依据会是**条件像控制信号还是像内容序列**，还是会被“某篇论文说过最好”带着跑？

这个问题，材料没有替你回答。  
但如果你已经能用刚才那个最小模型来拆它，你就不是只“听懂了 DiT”，而是开始会用了。
